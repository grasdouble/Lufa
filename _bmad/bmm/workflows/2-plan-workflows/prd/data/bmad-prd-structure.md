# BMAD PRD Standard Structure

This document defines the canonical structure for BMAD-compliant Product Requirements Documents.

## Standard Section Order

1. **Executive Summary**
   - High-level product overview
   - Key goals and vision

2. **Success Criteria**
   - User success metrics
   - Business success metrics

3. **Product Scope**
   - What's in scope
   - What's out of scope

4. **User Journeys**
   - User narratives and workflows
   - Key user interactions

5. **Domain Requirements** _(if applicable)_
   - Domain-specific considerations
   - Compliance requirements

6. **Innovation Analysis** _(if applicable)_
   - Novel approaches
   - Competitive differentiation

7. **Project-Type Requirements**
   - Requirements specific to project type (API, web app, mobile, etc.)
   - Technology-specific needs

8. **Functional Requirements**
   - Core capabilities
   - Feature requirements

9. **Non-Functional Requirements**
   - Performance, security, scalability
   - Quality attributes

## Formatting Standards

- Use ## Level 2 headers for main sections
- Ensure logical section ordering
- Maintain frontmatter compliance
