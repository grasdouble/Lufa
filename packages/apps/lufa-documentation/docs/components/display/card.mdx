# Card

A flexible container component for grouping related content with consistent styling and elevation.

## Overview

Cards are surface components that display content and actions about a single subject. The Card component provides:

- Clean, consistent container styling
- Optional elevated shadow for depth
- Interactive variant with hover effects
- Flexible composition with any content
- Responsive padding and spacing
- Accessible clickable areas when interactive

## Installation

The Card component is part of the Lufa Design System:

```tsx
import { Card } from "@grasdouble/lufa_design-system";
```

## Basic Usage

```tsx
<Card>
  <Typography variant="h3">Card Title</Typography>
  <Typography variant="body1">Card content goes here.</Typography>
</Card>
```

## Props

| Prop          | Type                  | Default | Description                               |
| ------------- | --------------------- | ------- | ----------------------------------------- |
| `children`    | `ReactNode`           | -       | Card content                              |
| `elevated`    | `boolean`             | `false` | Apply elevated shadow for more prominence |
| `interactive` | `boolean`             | `false` | Add hover effects and cursor pointer      |
| `onClick`     | `() => void`          | -       | Click handler (requires `interactive`)    |
| `className`   | `string`              | -       | Additional CSS classes                    |
| `style`       | `React.CSSProperties` | -       | Inline styles                             |

## Customization

### Custom Styles

You can customize cards using inline styles or CSS variables:

```tsx
<Card
  style={{
    backgroundColor: "var(--lufa-color-background-secondary)",
    border: "2px solid var(--lufa-color-primary)",
    padding: "var(--lufa-spacing-xl)",
  }}
>
  <Typography variant="h4">Custom Card</Typography>
</Card>
```

### Responsive Padding

```tsx
<Card
  style={{
    padding: "var(--lufa-spacing-md)",
    "@media (min-width: 768px)": {
      padding: "var(--lufa-spacing-xl)",
    },
  }}
>
  <Typography variant="h4">Responsive Card</Typography>
</Card>
```

## Accessibility

The Card component follows accessibility best practices:

- Uses semantic HTML elements
- Supports keyboard navigation when interactive
- Interactive cards have proper ARIA attributes
- Focus-visible indicator for keyboard users
- Adequate clickable area for interactive cards
- Screen reader announces clickable cards appropriately

## Best Practices

### Do

✅ Group related content together
✅ Use consistent padding and spacing
✅ Keep card content focused and concise
✅ Use cards to create visual hierarchy
✅ Maintain readable line length inside cards

### Don't

❌ Nest cards too deeply (max 1-2 levels)
❌ Overuse elevated or interactive variants
❌ Make cards too wide (maintain ~65-75 characters per line)
❌ Use cards for every piece of content
❌ Mix too many card styles on one page

## Design Tokens

Cards use the following design tokens:

```css
/* Background */
background-color: var(--lufa-color-background-primary);

/* Shadow */
box-shadow: var(--lufa-shadow-sm); /* default */
box-shadow: var(--lufa-shadow-md); /* elevated */

/* Spacing */
padding: var(--lufa-spacing-lg);
```
