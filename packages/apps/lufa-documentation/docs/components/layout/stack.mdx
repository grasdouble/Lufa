# Stack

A flexible layout component for arranging child elements in horizontal or vertical stacks with consistent spacing, alignment, and distribution options.

## Overview

The Stack component is a fundamental layout primitive that simplifies the creation of flexbox-based layouts. It provides a declarative API for controlling direction, spacing, alignment, wrapping, and justification of child elements. Stack is essential for building consistent, maintainable layouts throughout your application.

Key features:

- Horizontal and vertical layout directions
- Four gap sizes for consistent spacing
- Multiple alignment and justification options
- Wrapping control for responsive layouts
- Optional padding variants
- Composable with Stack.Item for advanced layouts


## Basic Usage

```tsx
// Vertical stack (default)
<Stack>
  <div>Item 1</div>
  <div>Item 2</div>
  <div>Item 3</div>
</Stack>

// Horizontal stack
<Stack direction={STACK_DIRECTION.horizontal}>
  <div>Item 1</div>
  <div>Item 2</div>
  <div>Item 3</div>
</Stack>
```

## Props

### Stack Props

| Prop        | Type                                                                | Default      | Description                                    |
| ----------- | ------------------------------------------------------------------- | ------------ | ---------------------------------------------- |
| `children`  | `ReactNode`                                                         | -            | Child elements to arrange in the stack         |
| `direction` | `'horizontal' \| 'vertical'`                                        | `'vertical'` | Layout direction of the stack                  |
| `gap`       | `'none' \| 'condensed' \| 'normal' \| 'spacious'`                   | `'normal'`   | Spacing between child elements                 |
| `align`     | `'stretch' \| 'start' \| 'center' \| 'end' \| 'baseline'`           | `'stretch'`  | Cross-axis alignment of children               |
| `justify`   | `'start' \| 'center' \| 'end' \| 'space-between' \| 'space-evenly'` | `'start'`    | Main-axis distribution of children             |
| `wrap`      | `'wrap' \| 'nowrap'`                                                | `'nowrap'`   | Whether children should wrap to multiple lines |
| `padding`   | `'none' \| 'condensed' \| 'normal' \| 'spacious'`                   | `'none'`     | Internal padding of the stack container        |

### Stack.Item Props

| Prop       | Type        | Default | Description                                          |
| ---------- | ----------- | ------- | ---------------------------------------------------- |
| `children` | `ReactNode` | -       | Content to display inside the stack item             |
| `grow`     | `boolean`   | `false` | Whether the item should grow to fill available space |

## States

Stack is a layout component without interactive states. It adapts to its children's states and maintains consistent layout regardless of child component states.

## Styling

The Stack component is built using Flexbox and includes:

### Base Styles

- Display: `flex`
- Flex direction: Controlled by `direction` prop
- Gap: Controlled by `gap` prop (0px, 8px, 16px, 24px)
- Alignment: Controlled by `align` prop
- Justification: Controlled by `justify` prop
- Wrapping: Controlled by `wrap` prop
- Padding: Controlled by `padding` prop

### CSS Classes

Stack uses CSS modules with the following class patterns:

- `gap-{size}`: Controls spacing between children
- `direction-{horizontal|vertical}`: Controls layout direction
- `align-{stretch|start|center|end|baseline}`: Controls cross-axis alignment
- `justify-{start|center|end|space-between|space-evenly}`: Controls main-axis distribution
- `wrap-{wrap|nowrap}`: Controls wrapping behavior
- `padding-{size}`: Controls internal padding

### Stack.Item Styles

- `grow`: Applies `flex-grow: 1` to fill available space

## Accessibility

Stack is a layout component with minimal accessibility requirements:

- **Semantic HTML**: Uses `<div>` elements (no semantic meaning)
- **No ARIA Required**: Pure layout component, doesn't need ARIA attributes
- **Keyboard Navigation**: Does not interfere with keyboard navigation of child components
- **Focus Management**: Maintains natural focus order of child elements
- **Screen Readers**: Transparent to screen readers, children are announced normally

### Best Practices for Accessibility

- Use semantic HTML elements within Stack (nav, main, aside, etc.)
- Ensure child components have proper ARIA attributes
- Maintain logical tab order by arranging children meaningfully
- Don't rely solely on visual spacing to convey relationships

## Best Practices

### Do ✅

- Use Stack for consistent spacing in layouts
- Nest Stacks to create complex, maintainable layouts
- Use semantic gap sizes (condensed, normal, spacious) instead of custom values
- Combine with Stack.Item for flexible layouts
- Use appropriate direction for content flow
- Choose wrap behavior based on responsive needs

### Don't ❌

- Don't use Stack when CSS Grid is more appropriate (e.g., complex 2D layouts)
- Don't nest excessively deep (consider composition patterns)
- Don't use Stack just for a single child
- Don't override Stack's flexbox styles with conflicting CSS
- Don't use inline styles for spacing (use gap prop instead)
