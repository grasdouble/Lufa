# Button

A versatile button component for triggering actions and navigation with multiple variants, sizes, and states.

## Overview

Buttons are interactive elements that users can click or tap to perform actions. The Button component provides:

- Multiple visual variants (primary, secondary, outline, ghost)
- Three size options for different contexts
- Built-in loading and disabled states
- Full accessibility support with keyboard navigation
- Flexible composition with icons and custom content

## Basic Usage

```tsx
<Button variant="primary">Primary Button</Button>
```

## Props

| Prop        | Type                                               | Default     | Description                       |
| ----------- | -------------------------------------------------- | ----------- | --------------------------------- |
| `variant`   | `'primary' \| 'secondary' \| 'outline' \| 'ghost'` | `'primary'` | Visual style variant              |
| `size`      | `'sm' \| 'md' \| 'lg'`                             | `'md'`      | Button size                       |
| `disabled`  | `boolean`                                          | `false`     | Disable button interactions       |
| `loading`   | `boolean`                                          | `false`     | Show loading state                |
| `fullWidth` | `boolean`                                          | `false`     | Make button full width            |
| `onClick`   | `(event: React.MouseEvent) => void`                | -           | Click handler                     |
| `type`      | `'button' \| 'submit' \| 'reset'`                  | `'button'`  | Button type attribute             |
| `children`  | `ReactNode`                                        | -           | Button content (text, icons, etc) |

### Form Submit

```tsx
<form onSubmit={handleSubmit}>
  <Input label="Name" />
  <Stack direction={STACK_DIRECTION.HORIZONTAL} gap="sm">
    <Button type="button" variant="ghost" onClick={handleReset}>
      Reset
    </Button>
    <Button type="submit" variant="primary">
      Submit
    </Button>
  </Stack>
</form>
```

### Async Action with Loading

```tsx
function SubmitButton() {
  const [loading, setLoading] = React.useState(false);

  const handleClick = async () => {
    setLoading(true);
    try {
      await submitForm();
    } finally {
      setLoading(false);
    }
  };

  return (
    <Button variant="primary" loading={loading} onClick={handleClick}>
      {loading ? "Submitting..." : "Submit"}
    </Button>
  );
}
```

## Accessibility

The Button component follows accessibility best practices:

- Uses semantic `<button>` element
- Supports keyboard navigation (Enter and Space keys)
- Properly handles disabled state (non-interactive, aria-disabled)
- Includes focus-visible indicator for keyboard users
- Loading state announced to screen readers
- Maintains adequate touch target size (44×44px minimum)

## Best Practices

### Do

✅ Use clear, action-oriented labels ("Save", "Delete", "Continue")
✅ Limit to one primary button per section
✅ Place primary action on the right in button groups
✅ Use consistent sizing within a context
✅ Provide visual feedback for interactive states

### Don't

❌ Use vague labels like "Click here" or "OK"
❌ Use too many primary buttons (creates confusion)
❌ Make buttons too small to tap on mobile (< 44px)
❌ Use buttons for navigation (use links instead)
❌ Disable buttons without explanation
