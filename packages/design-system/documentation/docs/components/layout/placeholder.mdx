# Placeholder

A versatile placeholder component used for layout spacing, visual structure, and prototyping. Features customizable height, width, and color variants with smooth transitions and modern styling.

## Overview

The Placeholder component is designed to help developers quickly prototype layouts, visualize component positioning, and maintain consistent spacing during development. It features a gradient background by default, customizable dimensions, and smooth hover effects.

## Live Demo

<LiveDemo />

## Installation

The Placeholder component is part of the Lufa Design System:

```tsx
import { Placeholder } from "@grasdouble/lufa_design-system";
```

## Basic Usage

```tsx
<Placeholder>Default Placeholder</Placeholder>
```

## Props

| Prop        | Type                                                 | Default    | Description                                  |
| ----------- | ---------------------------------------------------- | ---------- | -------------------------------------------- |
| `children`  | `ReactNode`                                          | -          | Content to display inside the placeholder    |
| `color`     | `string`                                             | -          | Solid background color (any valid CSS color) |
| `colorFrom` | `string`                                             | -          | Gradient start color (requires `colorTo`)    |
| `colorTo`   | `string`                                             | -          | Gradient end color (requires `colorFrom`)    |
| `height`    | `'small' \| 'medium' \| 'large' \| 'auto' \| 'full'` | `'medium'` | Height variant of the placeholder            |
| `width`     | `'auto' \| 'small' \| 'medium' \| 'large' \| 'full'` | `'auto'`   | Width variant of the placeholder             |

## Height Variants

The Placeholder component supports five height options:

<Height />

```tsx
// Small height (40px)
<Placeholder height="small">Small Height</Placeholder>

// Medium height (80px) - Default
<Placeholder height="medium">Medium Height</Placeholder>

// Large height (128px)
<Placeholder height="large">Large Height</Placeholder>

// Auto height (adjusts to content with min-height: 40px)
<Placeholder height="auto">
  Auto height adjusts to content
  <br />
  Multiple lines supported
</Placeholder>

// Full height (100% of parent container)
<Placeholder height="full">Full Height</Placeholder>
```

### Height Specifications

- **Small**: `40px` (h-10)
- **Medium**: `80px` (h-20) - Default
- **Large**: `128px` (h-32)
- **Auto**: Content-based with `min-height: 40px`
- **Full**: `100%` of parent container

## Width Variants

The Placeholder component supports five width options:

<Width />

```tsx
// Auto width (min-width: 96px)
<Placeholder width="auto">Auto</Placeholder>

// Small width (128px)
<Placeholder width="small">Small Width</Placeholder>

// Medium width (240px)
<Placeholder width="medium">Medium Width</Placeholder>

// Large width (384px)
<Placeholder width="large">Large Width</Placeholder>

// Full width (100% of parent container)
<Placeholder width="full">Full Width</Placeholder>
```

### Width Specifications

- **Auto**: Content-based with `min-width: 96px` - Default
- **Small**: `128px` (w-32)
- **Medium**: `240px` (w-60)
- **Large**: `384px` (w-96)
- **Full**: `100%` of parent container

## Custom Colors

Override the default violet gradient with any valid CSS color:

<CustomColors />

```tsx
import { semantic } from '@grasdouble/lufa_design-system';

// Default gradient
<Placeholder>Default Gradient</Placeholder>

// Solid colors using semantic tokens
<Placeholder color={tokens.color.interactive.default}>Solid Blue</Placeholder>
<Placeholder color={tokens.color.success.default}>Solid Green</Placeholder>
<Placeholder color={tokens.color.error.default}>Solid Red</Placeholder>
<Placeholder color={tokens.color.warning.default}>Solid Orange</Placeholder>

// Custom gradients
<Placeholder colorFrom={tokens.color.interactive.default} colorTo={tokens.color.brand.secondary}>
  Blue to Purple
</Placeholder>
<Placeholder colorFrom={tokens.color.success.default} colorTo="#14B8A6">
  Green to Teal
</Placeholder>
<Placeholder colorFrom={tokens.color.warning.default} colorTo={tokens.color.error.default}>
  Orange to Red
</Placeholder>

// Custom hex gradients
<Placeholder colorFrom="#3b82f6" colorTo="#8b5cf6">Blue to Purple</Placeholder>
<Placeholder colorFrom="#10b981" colorTo="#14B8A6">Green to Teal</Placeholder>
```

## Styling

The Placeholder component includes:

### Base Styles

- Flexbox display with centered content
- Large border radius (--radius-lg)
- Small shadow (--shadow-sm)
- Horizontal padding (16px)
- Small font size with medium weight
- White text color
- Smooth transitions (200ms)

### Default Gradient

```css
background: linear-gradient(to bottom right, violet-500, violet-600);
```

With hover state:

```css
background: linear-gradient(to bottom right, violet-600, violet-700);
```

### Custom Color Override

When a `color` prop is provided, it overrides the gradient with a solid color:

```tsx
<Placeholder color="#3b82f6">Solid blue background</Placeholder>
```

## Accessibility

The Placeholder component is primarily a development and prototyping tool:

- **Not for production UI**: Replace with actual components before shipping
- **No semantic meaning**: Uses a plain `div` element
- **No ARIA attributes**: Not intended for interactive use
- **Visual only**: Content is for developer reference during prototyping

If you need placeholder content in production (like skeleton screens), consider creating a dedicated loading component with proper accessibility attributes.

## Best Practices

### Do ✅

- Use for rapid prototyping and layout visualization
- Use color coding to distinguish different component types
- Combine with Stack for spacing demonstrations
- Replace with actual components before production

### Don't ❌

- Don't ship Placeholder components to production
- Don't use for interactive elements (use Button, Link, etc.)
- Don't rely on for final styling (implement proper components)
- Don't use as skeleton loading screens without accessibility improvements
