# Center

Centers children horizontally, vertically, or both using flex (optionally inline).

## Overview

Use `Center` when you need…

- Quick centering without custom flex CSS
- Axis control via `axis` (`horizontal`, `vertical`, `both`)
- Vertical centering via `minHeight`
- Inline flow support via `inline` (`inline-flex`)

## Import

```tsx
import { Center } from "@grasdouble/lufa_design-system";
```

## Basic Usage

```tsx
<Center axis="both" minHeight={240}>
  <div>Centered</div>
</Center>
```

## Props

| Prop        | Type                                   | Default  | Description                               |
| ----------- | -------------------------------------- | -------- | ----------------------------------------- |
| `axis`      | `'both' \| 'horizontal' \| 'vertical'` | `'both'` | Which axis to center on                   |
| `inline`    | `boolean`                              | `false`  | Uses `inline-flex` instead of `flex`      |
| `minHeight` | `number \| string`                     | -        | Optional minimum height for the container |
| `as`        | `'div' \| 'span' \| 'section' \| 'main' \| 'article'` | `'div'` | Render element                            |
| `children`  | `ReactNode`                            | -        | Content                                   |

Also supports standard element props (for example `className`, `style`, `aria-label`, …).

## Best Practices

### Do ✅

- Use `minHeight` when demonstrating or requiring vertical centering
- Use `inline` + `as="span"` when centering content inside text flow
- Prefer semantic `as` values (`main`, `section`, `article`) when the container has meaning

### Don't ❌

- Don’t use `Center` for complex layouts; use `Stack`/grid instead
- Don’t rely on `Center` to “fix” spacing issues; ensure container sizing is intentional

## Accessibility

- `Center` is purely layout; accessibility depends on its children.
- Use an appropriate `as` element for semantics when the container represents a meaningful region.
