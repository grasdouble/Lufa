# DatePicker

A date picker component that allows users to select dates using a native date input.

## Overview

Use `DatePicker` when you need to:

- Allow users to select dates from a calendar
- Enable date and time selection
- Enforce date ranges with min/max constraints
- Provide accessible date input

## Import

```tsx
import { DatePicker } from '@grasdouble/lufa_design-system';
```

## Basic Usage

```tsx
<DatePicker
  label="Select Date"
  onDateChange={(date) => console.log(date)}
/>
```

## Props

| Prop           | Type                                | Default      | Description                             |
| -------------- | ----------------------------------- | ------------ | --------------------------------------- |
| `label`        | `string`                            | -            | Label text above the input              |
| `size`         | `'small' \| 'medium' \| 'large'`    | `'medium'`   | Size of the input                       |
| `variant`      | `'outlined' \| 'filled'`            | `'outlined'` | Visual style variant                    |
| `fullWidth`    | `boolean`                           | `false`      | Makes the input take full container width|
| `showTime`     | `boolean`                           | `false`      | Enables time selection (datetime-local) |
| `onDateChange` | `(date: string) => void`            | -            | Callback when date changes              |
| `error`        | `string`                            | -            | Error message to display                |
| `helperText`   | `string`                            | -            | Helper text below the input             |
| `disabled`     | `boolean`                           | `false`      | Disables the input                      |
| `required`     | `boolean`                           | `false`      | Marks the field as required             |
| `min`          | `string`                            | -            | Minimum allowed date (YYYY-MM-DD)       |
| `max`          | `string`                            | -            | Maximum allowed date (YYYY-MM-DD)       |
| `value`        | `string`                            | -            | Controlled date value                   |
| `defaultValue` | `string`                            | -            | Default date value (uncontrolled)       |

## Best Practices

### Do ✅

- Use clear labels to indicate what date is being selected
- Provide min/max constraints when appropriate
- Use helper text to explain date requirements
- Set sensible default values when possible
- Handle date change callbacks appropriately

### Don't ❌

- Don't use for time-only selection (consider TimePicker)
- Avoid using without a label for accessibility
- Don't forget to validate date ranges
- Avoid overly restrictive date constraints without explanation

## Accessibility

- Uses native date input for broad support
- Label is properly associated with the input
- Keyboard accessible via Tab navigation and arrow keys
- Focus states are clearly visible
- Error states are announced to screen readers
- Calendar icon is decorative (pointer-events: none)

## Examples

### Date Only

```tsx
<DatePicker
  label="Date of Birth"
  max={new Date().toISOString().split('T')[0]}
  required
/>
```

### Date and Time

```tsx
<DatePicker
  label="Appointment DateTime"
  showTime={true}
  helperText="Select both date and time"
/>
```

### Date Range

```tsx
const [startDate, setStartDate] = useState('');
const [endDate, setEndDate] = useState('');

<>
  <DatePicker
    label="Start Date"
    onDateChange={setStartDate}
    max={endDate}
  />
  <DatePicker
    label="End Date"
    onDateChange={setEndDate}
    min={startDate}
  />
</>
```

### With Constraints

```tsx
<DatePicker
  label="Event Date"
  min="2024-01-01"
  max="2024-12-31"
  helperText="Select a date in 2024"
/>
```

### Form Integration

```tsx
<form onSubmit={handleSubmit}>
  <DatePicker
    label="Meeting Date"
    required
    fullWidth
    onDateChange={(date) => setFormData({ ...formData, meetingDate: date })}
  />
</form>
```
