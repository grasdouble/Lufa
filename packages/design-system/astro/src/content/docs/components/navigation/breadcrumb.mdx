# Breadcrumb

Hierarchical navigation trail that shows where you are in the page tree.

## Overview

Use `Breadcrumb` when you need…

- A clear location indicator in a page hierarchy
- Navigation back to parent pages
- Items via `items`
- Optional separator customization via `separator`
- Size control via `size`

## Import

```tsx

```

## Basic Usage

```tsx
<Breadcrumb
  items={[{ label: 'Home', href: '/' }, { label: 'Products', href: '/products' }, { label: 'Current Page' }]}
/>
```

## Props

| Prop        | Type                             | Default    | Description                    |
| ----------- | -------------------------------- | ---------- | ------------------------------ |
| `items`     | `BreadcrumbItem[]`               | -          | Array of breadcrumb items      |
| `separator` | `ReactNode`                      | `'/'`      | Custom separator between items |
| `size`      | `'small' \| 'medium' \| 'large'` | `'medium'` | Size variant                   |
| `className` | `string`                         | -          | Additional CSS class           |

### BreadcrumbItem

| Property | Type        | Description                               |
| -------- | ----------- | ----------------------------------------- |
| `label`  | `string`    | Label to display                          |
| `href`   | `string`    | Link URL (optional for current/last item) |
| `icon`   | `ReactNode` | Icon before label (optional)              |

## Best Practices

### Do ✅

- Keep labels short and meaningful
- Use the last item as the current page (no `href`)
- Use `size="small"` in dense headers or tight layouts

### Don't ❌

- Don’t include too many levels; consider truncation or simplifying IA
- Don’t use breadcrumbs as the only navigation for critical flows

## Accessibility

- `Breadcrumb` renders a `<nav aria-label="Breadcrumb">` and marks the current page with `aria-current="page"`.
- Ensure link labels are descriptive and unique enough in context.
