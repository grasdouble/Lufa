# Button

<DarkModeCompatible />

A versatile button component for actions and navigation. Supports multiple variants, colors, sizes, icons, and state indicators for loading and disabled states.

## Overview

Use `Button` when you need…

- A clear action trigger (primary/secondary/tertiary)
- Styling via `variant` and `color`
- Sizing via `size` and layout via `fullWidth`
- Async states via `loading` and `disabled`
- Icons via `startIcon` / `endIcon`

## Anatomy

The Button component consists of several key parts:

```
┌─────────────────────────────────────┐
│  [startIcon]  Label  [endIcon]      │  ← Button Container
└─────────────────────────────────────┘
     ↑          ↑         ↑
     │          │         └─ Optional trailing icon
     │          └─────────── Required text content
     └──────────────────────── Optional leading icon
```

**Component Structure:**

- **Container**: Handles layout, spacing, colors, and interaction states
- **Start Icon** (optional): Leading icon positioned before the label
- **Label**: The main text content (required for accessibility)
- **End Icon** (optional): Trailing icon positioned after the label

**Visual States:**

- Default: Normal interactive state
- Hover: Visual feedback on mouse over
- Focus: Keyboard navigation indicator (focus ring)
- Active: Pressed/clicking state
- Loading: Spinner replaces content, interaction disabled
- Disabled: Reduced opacity, no interaction

## API Reference

### Props

| Prop        | Type                                                             | Default     | Description                                    |
| ----------- | ---------------------------------------------------------------- | ----------- | ---------------------------------------------- |
| `children`  | `ReactNode`                                                      | Required    | Button content (text, icons, etc.)             |
| `label`     | `string`                                                         | -           | **Deprecated** (use `children`)                |
| `variant`   | `'solid' \| 'outlined' \| 'text' \| 'ghost' \| 'link'`           | `'solid'`   | Visual style variant                           |
| `color`     | `'primary' \| 'secondary' \| 'success' \| 'warning' \| 'danger'` | `'primary'` | Color scheme                                   |
| `size`      | `'small' \| 'medium' \| 'large'`                                 | `'medium'`  | Button size                                    |
| `fullWidth` | `boolean`                                                        | `false`     | Expands button to full container width         |
| `loading`   | `boolean`                                                        | `false`     | Shows loading spinner and disables interaction |
| `disabled`  | `boolean`                                                        | `false`     | Disables button interaction                    |
| `startIcon` | `ReactNode`                                                      | -           | Icon displayed before label                    |
| `endIcon`   | `ReactNode`                                                      | -           | Icon displayed after label                     |
| `onClick`   | `(event: React.MouseEvent<HTMLButtonElement>) => void`           | -           | Click event handler                            |
| `type`      | `'button' \| 'submit' \| 'reset'`                                | `'button'`  | HTML button type attribute                     |

Also supports all standard HTML `button` attributes (e.g., `className`, `id`, `aria-*`, `data-*`).

### Variant Details

- **solid**: Filled background with high emphasis (default for primary actions)
- **outlined**: Border with transparent background (secondary actions)
- **text**: No border or background, minimal visual weight
- **ghost**: Similar to text but with hover background
- **link**: Appears as a text link (useful for inline navigation)

### Color Details

Each color maps to semantic design tokens:

- **primary**: Main brand color, use for primary actions
- **secondary**: Neutral alternative, use for secondary actions
- **success**: Positive actions (e.g., "Confirm", "Save")
- **warning**: Cautionary actions (e.g., "Archive", "Unpublish")
- **danger**: Destructive actions (e.g., "Delete", "Remove")

### Size Details

- **small**: Compact size for dense interfaces (32px height)
- **medium**: Default size for most use cases (40px height)
- **large**: Prominent size for hero sections (48px height)

## Best Practices

### Do ✅

- Use a single clear primary action per view when possible
- Use `loading` for async actions to prevent double-submits
- Keep labels short and verb-first (“Save”, “Continue”, “Delete”)
- Prefer `type="submit"` inside forms for submission actions

### Don't ❌

- Don’t disable a button without explaining why (use helper text/tooltip if needed)
- Don’t use `link` variant for destructive actions; use an appropriate `color`

## Accessibility

The Button component is designed with WCAG 2.1 AA compliance in mind:

### Keyboard Navigation

- **Tab**: Navigate to button
- **Enter/Space**: Activate button
- **Escape**: Cancel action (in dialogs or modals)

All buttons are keyboard accessible by default with visible focus indicators.

### Screen Reader Support

**For Buttons with Text:**

```tsx
<Button>Save Changes</Button>
// Screen reader announces: "Save Changes, button"
```

**For Icon-Only Buttons:**

```tsx
<Button aria-label="Close dialog">
  <CloseIcon />
</Button>
// Screen reader announces: "Close dialog, button"
```

**For Loading State:**

```tsx
<Button loading aria-label="Saving changes">
  Save
</Button>
// Screen reader announces: "Saving changes, button, busy"
```

### Best Practices for Accessibility

#### Do ✅

- Always provide meaningful button labels
- Use `aria-label` for icon-only buttons
- Use `loading` prop with descriptive `aria-label` for async actions
- Ensure sufficient color contrast (all variants meet WCAG AA)
- Keep focus indicators visible (default focus ring)

#### Don't ❌

- Don't use only color to convey meaning (combine with icons/text)
- Don't remove focus outlines (breaks keyboard navigation)
- Don't nest interactive elements inside buttons
- Don't use `disabled` without explanation (add helper text if needed)

### ARIA Attributes

The component automatically handles:

- `aria-disabled="true"` when `disabled` or `loading`
- `aria-busy="true"` when `loading`

Additional ARIA attributes can be passed as props:

```tsx
<Button aria-label="Delete item" aria-describedby="delete-warning">
  Delete
</Button>
```

### Color Contrast

All button variants meet WCAG 2.1 AA contrast requirements:

- Normal text: Minimum 4.5:1 contrast ratio
- Large text: Minimum 3:1 contrast ratio
- Interactive elements: Minimum 3:1 contrast ratio

Tested in both light and dark modes.

## Common Patterns

### Form Submission

```tsx title="src/components/LoginForm.tsx" {8,16-18}
import { useState } from 'react';

import { Button } from '@grasdouble/lufa_design-system';

function LoginForm() {
  const [loading, setLoading] = useState(false);

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();
    setLoading(true);
    try {
      await login();
    } finally {
      setLoading(false);
    }
  };

  return (
    <form onSubmit={handleSubmit}>
      {/* form fields */}
      <Button type="submit" loading={loading}>
        Sign In
      </Button>
    </form>
  );
}
```

### Confirmation Dialog

```tsx title="src/components/DeleteConfirmation.tsx"
import { Button, Stack } from '@grasdouble/lufa_design-system';

function DeleteConfirmation({ onConfirm, onCancel }) {
  return (
    <Stack direction="row" spacing="medium" justify="end">
      <Button variant="outlined" onClick={onCancel}>
        Cancel
      </Button>
      <Button color="danger" onClick={onConfirm}>
        Delete
      </Button>
    </Stack>
  );
}
```

### Navigation Button

```tsx title="src/components/Navigation.tsx"
import { useNavigate } from 'react-router-dom';

import { Button } from '@grasdouble/lufa_design-system';

function Navigation() {
  const navigate = useNavigate();

  return (
    <Button variant="link" onClick={() => navigate('/dashboard')}>
      Go to Dashboard
    </Button>
  );
}
```

### Async Action with Feedback

```tsx title="src/components/SaveButton.tsx"
import { useState } from 'react';

import { Button } from '@grasdouble/lufa_design-system';

function SaveButton() {
  const [saving, setSaving] = useState(false);

  const handleSave = async () => {
    setSaving(true);
    try {
      await saveData();
      // Show success message
    } catch (error) {
      // Show error message
    } finally {
      setSaving(false);
    }
  };

  return (
    <Button loading={saving} onClick={handleSave} aria-label={saving ? 'Saving changes' : 'Save changes'}>
      {saving ? 'Saving...' : 'Save Changes'}
    </Button>
  );
}
```

## TypeScript Support

The Button component is fully typed with TypeScript:

```tsx
import type { ComponentPropsWithoutRef } from 'react';

interface ButtonProps extends ComponentPropsWithoutRef<'button'> {
  variant?: 'solid' | 'outlined' | 'text' | 'ghost' | 'link';
  color?: 'primary' | 'secondary' | 'success' | 'warning' | 'danger';
  size?: 'small' | 'medium' | 'large';
  fullWidth?: boolean;
  loading?: boolean;
  startIcon?: ReactNode;
  endIcon?: ReactNode;
}
```

This ensures type safety and autocomplete in your IDE.
