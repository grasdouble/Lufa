# Modal

<DarkModeCompatible />

A modal dialog component that displays content overlaid on the main page. Supports multiple sizes, customizable headers and footers, and flexible close behaviors.

## Overview

Use `Modal` when you need…

- To focus user attention on a specific task or message
- To collect information without navigating away
- Confirmation dialogs for destructive actions
- Complex forms or multi-step workflows
- Displaying detailed content without page navigation
- Configurable sizes from small alerts to fullscreen
- Keyboard and backdrop close behaviors

## Basic Usage

```tsx title="src/App.tsx"
import { Modal, Button } from '@grasdouble/lufa_design-system';
import { useState } from 'react';

function App() {
  const [isOpen, setIsOpen] = useState(false);

  return (
    <>
      <Button onClick={() => setIsOpen(true)}>Open Modal</Button>

      <Modal
        open={isOpen}
        onClose={() => setIsOpen(false)}
        title="Modal Title"
      >
        <p>Modal content goes here.</p>
      </Modal>
    </>
  );
}
```

## Anatomy

The Modal component consists of several key parts:

```
┌────────────────────────────────────────┐
│  Backdrop (semi-transparent overlay)   │
│                                        │
│  ┌──────────────────────────────────┐ │
│  │  ┌──────────────────────────┐   │ │
│  │  │ Title            [Close] │   │ │ ← Header
│  │  └──────────────────────────┘   │ │
│  │  ┌──────────────────────────┐   │ │
│  │  │                          │   │ │
│  │  │  Modal Content           │   │ │ ← Content
│  │  │                          │   │ │
│  │  └──────────────────────────┘   │ │
│  │  ┌──────────────────────────┐   │ │
│  │  │  [Cancel]  [Confirm]     │   │ │ ← Footer (optional)
│  │  └──────────────────────────┘   │ │
│  └──────────────────────────────────┘ │
│            ↑ Modal Dialog             │
└────────────────────────────────────────┘
```

**Component Structure:**
- **Backdrop**: Semi-transparent overlay that dims the background content
- **Modal Dialog**: The main container centered on the screen
- **Header** (optional): Contains title and close button
- **Title** (optional): Descriptive heading for the modal
- **Close Button** (optional): X icon button in the header for closing
- **Content**: The main body area for modal content
- **Footer** (optional): Action buttons or additional content at the bottom

**Visual States:**
- Closed: Not rendered (display: none)
- Opening: Fade-in animation (backdrop and modal)
- Open: Fully visible, body scroll locked
- Closing: Fade-out animation

**Interaction Behaviors:**
- Backdrop click: Closes modal (configurable via `closeOnBackdropClick`)
- Escape key: Closes modal (configurable via `closeOnEscape`)
- Close button: Always closes modal when clicked
- Body scroll: Locked when modal is open

## API Reference

### Props

| Prop                      | Type                                            | Default   | Description                                   |
| ------------------------- | ----------------------------------------------- | --------- | --------------------------------------------- |
| `children`                | `ReactNode`                                     | Required  | Modal content                                 |
| `open`                    | `boolean`                                       | Required  | Controls modal visibility                     |
| `onClose`                 | `() => void`                                    | Required  | Close handler callback                        |
| `title`                   | `string`                                        | -         | Modal title displayed in header               |
| `size`                    | `'small' \| 'medium' \| 'large' \| 'fullscreen'` | `'medium'` | Modal width/size                             |
| `footer`                  | `ReactNode`                                     | -         | Footer content (buttons, actions)             |
| `closeOnBackdropClick`    | `boolean`                                       | `true`    | Close modal when clicking backdrop            |
| `closeOnEscape`           | `boolean`                                       | `true`    | Close modal when pressing Escape key          |
| `className`               | `string`                                        | -         | Additional CSS classes for modal dialog       |

### Size Details

- **small**: Compact modal for simple alerts or confirmations (400px max-width)
- **medium**: Default size for most use cases (600px max-width)
- **large**: Wider modal for forms or detailed content (800px max-width)
- **fullscreen**: Full viewport coverage for immersive experiences

### Close Behaviors

The modal provides two configurable close triggers:

- **closeOnBackdropClick**: When `true` (default), clicking the semi-transparent backdrop closes the modal. Set to `false` for critical actions that require explicit user decision.
- **closeOnEscape**: When `true` (default), pressing the Escape key closes the modal. Set to `false` to require explicit button interaction.

## Best Practices

### Do ✅

- Always provide a `title` for screen reader users
- Include a clear way to close the modal (close button, Cancel button, or backdrop click)
- Use `size="small"` for simple confirmations
- Use `size="large"` or `size="fullscreen"` for complex forms
- Lock body scroll when modal is open (handled automatically)
- Set `closeOnBackdropClick={false}` for critical actions
- Use clear action buttons in footer ("Cancel" + "Confirm")
- Keep modal content focused and concise

### Don't ❌

- Don't nest modals (modal within modal)
- Don't disable both close methods (`closeOnBackdropClick={false}` and `closeOnEscape={false}`) without providing close button
- Don't use modals for simple notifications (use Alert or Toast instead)
- Don't put too much content in modals (use fullscreen size or separate page)
- Don't auto-open modals on page load (anti-pattern)
- Don't forget to handle the `onClose` callback

## Accessibility

The Modal component is designed with WCAG 2.1 AA compliance in mind:

### Keyboard Navigation

- **Escape**: Close modal (unless `closeOnEscape={false}`)
- **Tab**: Navigate forward through focusable elements
- **Shift + Tab**: Navigate backward through focusable elements
- **Enter/Space**: Activate buttons and interactive elements

The modal automatically:
- Traps focus within the modal when open
- Locks body scroll to prevent background scrolling
- Returns focus to trigger element when closed

### Screen Reader Support

**For Standard Modals:**
```tsx
<Modal open={isOpen} onClose={handleClose} title="Confirm Action">
  <p>Are you sure you want to proceed?</p>
</Modal>
// Screen reader announces: "Confirm Action, dialog"
```

**For Alert Modals:**
```tsx
<Modal
  open={isOpen}
  onClose={handleClose}
  title="Error"
  role="alertdialog"
>
  <p>An error occurred.</p>
</Modal>
// Screen reader announces: "Error, alert dialog"
```

**Close Button:**
```tsx
// The close button has aria-label="Close" by default
// Screen reader announces: "Close, button"
```

### Best Practices for Accessibility

#### Do ✅

- Always provide a `title` for context
- Use semantic HTML in modal content
- Include visible focus indicators on interactive elements
- Provide clear action labels in footer ("Save", "Cancel", not just "OK")
- Use `role="alertdialog"` for critical messages
- Return focus to trigger element after closing
- Keep focus trapped within modal

#### Don't ❌

- Don't open modals without user action (auto-open on page load)
- Don't remove focus indicators
- Don't disable keyboard navigation
- Don't nest modals (confusing for screen readers)
- Don't use only color to convey information

### ARIA Attributes

The component automatically handles:
- `role="dialog"` on the modal container
- `aria-modal="true"` to indicate modal behavior
- `aria-labelledby` pointing to the title element (when title is provided)
- Focus trap within modal boundaries

Additional ARIA attributes can be passed via props:

```tsx
<Modal
  open={isOpen}
  onClose={handleClose}
  title="Confirmation"
  role="alertdialog"
  aria-describedby="modal-description"
>
  <p id="modal-description">This action cannot be undone.</p>
</Modal>
```

### Color Contrast

All modal elements meet WCAG 2.1 AA contrast requirements:
- Title text: Minimum 4.5:1 contrast ratio
- Body text: Minimum 4.5:1 contrast ratio
- Buttons: Minimum 3:1 contrast ratio
- Backdrop: Semi-transparent overlay ensuring clear distinction from background

Tested in both light and dark modes.

## Common Patterns

### Confirmation Dialog

```tsx title="src/components/DeleteConfirmation.tsx"
import { Modal, Button, Stack } from '@grasdouble/lufa_design-system';
import { useState } from 'react';

function DeleteConfirmation({ itemName, onConfirm }) {
  const [isOpen, setIsOpen] = useState(false);
  const [isDeleting, setIsDeleting] = useState(false);

  const handleConfirm = async () => {
    setIsDeleting(true);
    try {
      await onConfirm();
      setIsOpen(false);
    } finally {
      setIsDeleting(false);
    }
  };

  return (
    <>
      <Button color="danger" onClick={() => setIsOpen(true)}>
        Delete
      </Button>

      <Modal
        open={isOpen}
        onClose={() => setIsOpen(false)}
        title="Confirm Deletion"
        size="small"
        closeOnBackdropClick={false}
        footer={
          <Stack direction="row" spacing="medium" justify="end">
            <Button variant="outlined" onClick={() => setIsOpen(false)}>
              Cancel
            </Button>
            <Button color="danger" onClick={handleConfirm} loading={isDeleting}>
              Delete
            </Button>
          </Stack>
        }
      >
        <p>Are you sure you want to delete "{itemName}"? This action cannot be undone.</p>
      </Modal>
    </>
  );
}
```

### Form Modal

```tsx title="src/components/CreateUserModal.tsx"
import { Modal, Input, Button, Stack } from '@grasdouble/lufa_design-system';
import { useState } from 'react';

function CreateUserModal({ open, onClose, onSubmit }) {
  const [formData, setFormData] = useState({ name: '', email: '' });
  const [errors, setErrors] = useState<{ name?: string; email?: string }>({});
  const [isSubmitting, setIsSubmitting] = useState(false);

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();
    setIsSubmitting(true);
    try {
      await onSubmit(formData);
      onClose();
      setFormData({ name: '', email: '' });
    } finally {
      setIsSubmitting(false);
    }
  };

  return (
    <Modal
      open={open}
      onClose={onClose}
      title="Create New User"
      size="medium"
      footer={
        <Stack direction="row" spacing="medium" justify="end">
          <Button variant="outlined" onClick={onClose}>
            Cancel
          </Button>
          <Button type="submit" form="create-user-form" loading={isSubmitting}>
            Create User
          </Button>
        </Stack>
      }
    >
      <form id="create-user-form" onSubmit={handleSubmit}>
        <Stack direction="column" spacing="medium">
          <Input
            label="Name"
            value={formData.name}
            onChange={(e) => setFormData({ ...formData, name: e.target.value })}
            error={errors.name}
            required
            fullWidth
          />
          <Input
            label="Email"
            type="email"
            value={formData.email}
            onChange={(e) => setFormData({ ...formData, email: e.target.value })}
            error={errors.email}
            required
            fullWidth
          />
        </Stack>
      </form>
    </Modal>
  );
}
```

### Multi-Step Wizard

```tsx title="src/components/WizardModal.tsx"
import { Modal, Button, Stack } from '@grasdouble/lufa_design-system';
import { useState } from 'react';

function WizardModal({ open, onClose }) {
  const [step, setStep] = useState(1);
  const totalSteps = 3;

  const handleNext = () => setStep((prev) => Math.min(prev + 1, totalSteps));
  const handleBack = () => setStep((prev) => Math.max(prev - 1, 1));
  const handleFinish = () => {
    // Handle completion
    onClose();
    setStep(1);
  };

  return (
    <Modal
      open={open}
      onClose={onClose}
      title={`Setup Wizard - Step ${step} of ${totalSteps}`}
      size="large"
      closeOnBackdropClick={false}
      footer={
        <Stack direction="row" spacing="medium" justify="space-between">
          <Button variant="outlined" onClick={onClose}>
            Cancel
          </Button>
          <Stack direction="row" spacing="medium">
            {step > 1 && (
              <Button variant="outlined" onClick={handleBack}>
                Back
              </Button>
            )}
            {step < totalSteps ? (
              <Button onClick={handleNext}>Next</Button>
            ) : (
              <Button onClick={handleFinish}>Finish</Button>
            )}
          </Stack>
        </Stack>
      }
    >
      {step === 1 && <div>Step 1 content</div>}
      {step === 2 && <div>Step 2 content</div>}
      {step === 3 && <div>Step 3 content</div>}
    </Modal>
  );
}
```

### Fullscreen Modal

```tsx title="src/components/ImageGalleryModal.tsx"
import { Modal, Button } from '@grasdouble/lufa_design-system';

function ImageGalleryModal({ open, onClose, images }) {
  return (
    <Modal
      open={open}
      onClose={onClose}
      title="Image Gallery"
      size="fullscreen"
      footer={
        <Button onClick={onClose} variant="outlined">
          Close Gallery
        </Button>
      }
    >
      <div style={{ display: 'grid', gridTemplateColumns: 'repeat(auto-fill, minmax(200px, 1fr))', gap: '16px' }}>
        {images.map((img, idx) => (
          <img key={idx} src={img} alt={`Gallery image ${idx + 1}`} style={{ width: '100%', height: 'auto' }} />
        ))}
      </div>
    </Modal>
  );
}
```
