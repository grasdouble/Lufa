# Badge

<DarkModeCompatible />

A compact component for displaying status labels, notification counts, tags, and metadata.

## Overview

The Badge component provides a visual indicator for statuses, counts, categories, or other metadata. It's designed to be small, unobtrusive, and semantically meaningful through color-coded variants.

**When to use Badge:**

- **Status indicators**: Show active, pending, error, or warning states
- **Notification counts**: Display number of unread messages, alerts, or updates (e.g., "3 new messages")
- **Category tags**: Label content with categories or topics (e.g., "React", "TypeScript", "Bug")
- **Version labels**: Indicate software versions or feature states (e.g., "v2.0", "Beta", "New")
- **Metadata display**: Show counts, types, or attributes inline with content

**When NOT to use Badge:**

- For actionable elements (use Button instead)
- For large blocks of text (use Text component)
- For complex status information (use custom components)

## Anatomy

The Badge component consists of three main parts:

```
┌─────────────────────────┐
│ ● Content               │  ← Badge container (span by default)
│   │   │                 │
│   │   └─ Text content   │
│   └─ Dot indicator      │
│      (optional)          │
└─────────────────────────┘
```

**Parts:**

1. **Container**: Wraps the entire badge (default: `<span>`, customizable via `as` prop)
2. **Dot indicator**: Optional visual indicator (6px circle) for notifications or emphasis
3. **Content**: Text or children content (required)

**CSS Classes:**

- `.badge`: Base class applied to all badges
- `.variant-{name}`: Variant-specific styling (default, success, error, warning, info)
- `.size-{name}`: Size-specific styling (sm, md, lg)
- `.badge-with-dot`: Applied when `dot={true}` for spacing adjustment
- `.badge-dot`: The dot indicator element
- `.badge-content`: Content wrapper when dot is present

## API Reference

### Props

| Prop        | Type                                                       | Default      | Description                                                |
| ----------- | ---------------------------------------------------------- | ------------ | ---------------------------------------------------------- |
| `variant`   | `'default' \| 'success' \| 'error' \| 'warning' \| 'info'` | `'default'`  | Semantic variant defining color scheme and meaning         |
| `size`      | `'sm' \| 'md' \| 'lg'`                                     | `'md'`       | Size of the badge affecting padding and font size          |
| `dot`       | `boolean`                                                  | `false`      | Shows notification dot indicator before content            |
| `children`  | `React.ReactNode`                                          | **required** | Content to display inside the badge (text, numbers, etc.)  |
| `className` | `string`                                                   | -            | Additional CSS classes to apply                            |
| `as`        | `ElementType`                                              | `'span'`     | HTML element type to render (`span`, `div`, `label`, etc.) |
| `...rest`   | -                                                          | -            | All other props are passed to the underlying element       |

### Dot Indicator

The `dot` prop adds a small circular indicator (6px) before the badge content. This is useful for drawing attention to notifications, unread items, or emphasized statuses.

**When to use dot:**

- Notification counts: `<Badge dot variant="error">3 new</Badge>`
- Unread indicators: `<Badge dot>Unread</Badge>`
- Active status with emphasis: `<Badge dot variant="success">Online</Badge>`

**Properties:**

- Size: 6px diameter
- Position: Left side before content
- Spacing: Automatically adjusts content padding
- Accessibility: Hidden from screen readers (`aria-hidden="true"`)

```tsx
{/* Notification badges with dot */}
<Badge dot variant="error">5 alerts</Badge>
<Badge dot variant="warning">2 pending</Badge>
<Badge dot variant="success">Online</Badge>

{/* Without dot for comparison */}
<Badge variant="error">5 alerts</Badge>
<Badge variant="warning">2 pending</Badge>
<Badge variant="success">Online</Badge>
```

### Polymorphic Rendering (`as` prop)

The Badge component supports polymorphic rendering via the `as` prop. This allows you to render the badge as different HTML elements while preserving all styling.

**as="span" (default)**

- Renders as `<span>` element
- Use for inline text, general-purpose badges
- Inline display by default

```tsx
<Badge as="span">Default Span</Badge>
```

**as="div"**

- Renders as `<div>` element
- Use when block-level element is needed
- Inherits inline-flex display from badge styles

```tsx
<Badge as="div">Block Badge</Badge>
```

**as="label"**

- Renders as `<label>` element
- Use for form-related labels with badge styling
- Supports `htmlFor` and other label-specific props

```tsx
<Badge as="label" htmlFor="input-id" variant="info">
  Required Field
</Badge>
<input id="input-id" type="text" />
```

**Type Safety**

- TypeScript automatically infers correct props based on `as` value
- When `as="label"`, label-specific props (`htmlFor`) are available
- When `as="div"`, div-specific props are available

## Best Practices

### Do ✅

- Use semantic variants to communicate meaning (success = positive, error = negative, etc.)
- Keep badge text short (1-3 words maximum, or just numbers)
- Use `dot` prop to draw attention to notifications or important updates
- Combine badge with Text or other components for context
- Use consistent variant meanings across your application
- Provide text content (not just color) for accessibility
- Use `size="sm"` for inline tags within text blocks
- Use `size="lg"` for prominent standalone status indicators
- Test badges in both light and dark modes (all variants are dark mode compatible)

### Don't ❌

- Don't use badges for actionable elements (use Button instead)
- Don't use too many badge variants on one screen (creates visual noise)
- Don't rely solely on color to convey meaning (always include text)
- Don't use long text in badges (use Text or other components instead)
- Don't overuse the `dot` indicator (loses emphasis when overused)
- Don't use badges for multi-line content (single-line only)
- Don't use badges as primary navigation elements
- Don't use custom colors via inline styles (use the variant system)
- Don't make badges interactive (no onClick handlers - use Button for actions)
- Don't use badges for critical information without accompanying context

## Accessibility

The Badge component is designed with WCAG 2.1 AA compliance and includes accessibility considerations.

### Semantic HTML

**Default Rendering:**

- Badges render as `<span>` by default (inline element)
- Inline elements integrate naturally with text flow
- Screen readers announce badge content as part of surrounding text

```tsx
<p>
  Status: <Badge variant="success">Active</Badge>
</p>
// Screen reader: "Status: Active"
```

**Label Rendering:**

```tsx
<Badge as="label" htmlFor="email" variant="info">
  Email Address
</Badge>
<input id="email" type="email" />
// Screen reader associates label with input
```

### Screen Reader Support

**Basic Badge:**

```tsx
<Badge>New</Badge>
// Screen reader announces: "New"
```

**Badge with Variant:**

```tsx
<Badge variant="success">Active</Badge>
// Screen reader announces: "Active"
// Note: Color alone doesn't convey meaning - text is required
```

**Badge with Dot Indicator:**

```tsx
<Badge dot variant="error">
  3 notifications
</Badge>
// Screen reader announces: "3 notifications"
// Dot is decorative (aria-hidden="true") and not announced
```

### Best Practices for Accessibility

#### Do ✅

- Always include text content (not just color indicators)
- Use semantic variants that match text meaning (error variant for "Error" text)
- Provide context for badges (don't use standalone without explanation)
- Ensure badge text is concise and descriptive
- Test with screen readers (NVDA, JAWS, VoiceOver)
- Use proper element types (`as="label"` for form labels)
- Combine badges with aria-labels on parent elements when needed

#### Don't ❌

- Don't rely solely on color to convey meaning (fails WCAG)
- Don't use badges without text content (color-only badges are not accessible)
- Don't use vague text like "Status" without specifying what status
- Don't use badges for critical information without context
- Don't make badges the only way to convey important information

### ARIA Attributes

**Manual ARIA attributes (when needed):**

```tsx
{/* Badge with context from aria-label on parent */}
<div aria-label="Order status: Pending">
  <Badge variant="warning">Pending</Badge>
</div>

{/* Badge with additional description */}
<Badge variant="error" aria-describedby="error-description">
  Error
</Badge>
<span id="error-description" className="sr-only">
  Failed to load data. Please try again.
</span>
```

**Dot indicator:**

- Automatically has `aria-hidden="true"` (decorative element)
- Does not convey information to screen readers
- Text content must provide all necessary information

```tsx
{
  /* ✅ Good - text provides full context */
}
<Badge dot variant="error">
  3 unread notifications
</Badge>;

{
  /* ❌ Bad - text doesn't explain what dot means */
}
<Badge dot variant="error">
  3
</Badge>;
```

### Color Contrast

All Badge variants meet **WCAG 2.1 AA** contrast requirements:

- **Text contrast**: Minimum 4.5:1 ratio (text to background)
- **Component contrast**: Minimum 3:1 ratio (badge to page background)
- **Dark mode**: All variants maintain contrast ratios in dark mode

**Tested in both light and dark modes**. All semantic color tokens automatically adapt while maintaining accessibility.

```tsx
{
  /* ✅ WCAG AA compliant: success variant */
}
<Badge variant="success">Active</Badge>;

{
  /* ✅ WCAG AA compliant: error variant */
}
<Badge variant="error">Failed</Badge>;

{
  /* ✅ WCAG AA compliant: info variant */
}
<Badge variant="info">Beta</Badge>;
```

## Common Patterns

### Pattern 1: Status Labels in Lists

Display status indicators in data tables or lists.

```tsx title="src/components/UserList.tsx"
import { Badge } from '@grasdouble/lufa_design-system';

interface User {
  id: string;
  name: string;
  status: 'active' | 'inactive' | 'pending';
}

function UserList({ users }: { users: User[] }) {
  const getStatusBadge = (status: User['status']) => {
    switch (status) {
      case 'active':
        return <Badge variant="success">Active</Badge>;
      case 'inactive':
        return <Badge variant="error">Inactive</Badge>;
      case 'pending':
        return <Badge variant="warning">Pending</Badge>;
      default:
        return <Badge variant="default">Unknown</Badge>;
    }
  };

  return (
    <ul>
      {users.map((user) => (
        <li key={user.id} style={{ display: 'flex', alignItems: 'center', gap: '12px', padding: '12px' }}>
          <span>{user.name}</span>
          {getStatusBadge(user.status)}
        </li>
      ))}
    </ul>
  );
}
```

## TypeScript Support

The Badge component provides comprehensive TypeScript support with full type safety.

### Basic Usage with Types

```tsx
import type { BadgeProps } from '@grasdouble/lufa_design-system';
import { Badge } from '@grasdouble/lufa_design-system';

// Component with explicit props type
function StatusBadge({ status }: { status: string }) {
  return <Badge variant="success">{status}</Badge>;
}

// Props interface extending BadgeProps
interface CustomBadgeProps extends BadgeProps {
  count: number;
}

function NotificationBadge({ count, ...badgeProps }: CustomBadgeProps) {
  return (
    <Badge dot variant="error" {...badgeProps}>
      {count} new
    </Badge>
  );
}
```

### Polymorphic Type Safety

When using the `as` prop, TypeScript automatically infers the correct props for the element type:

```tsx
// As span (default) - HTMLSpanElement props available
<Badge as="span" title="Tooltip text">Span Badge</Badge>

// As div - HTMLDivElement props available
<Badge as="div" role="status">Div Badge</Badge>

// As label - HTMLLabelElement props available
<Badge as="label" htmlFor="input-id">Label Badge</Badge>

// TypeScript error: htmlFor not valid for span
<Badge as="span" htmlFor="input-id">Error</Badge>
//                ^^^^^^^ Error: Property 'htmlFor' does not exist
```

### Variant Type Guards

```tsx
import type { BadgeProps } from '@grasdouble/lufa_design-system';

// Type-safe variant selection
type StatusVariant = Extract<BadgeProps['variant'], 'success' | 'error' | 'warning'>;

    case 'failed':

// Usage with type safety
const status: 'active' = 'active';
<Badge variant={getStatusVariant(status)}>Status</Badge>;
```

## Troubleshooting

### Badge Not Displaying Correctly

**Problem**: Badge appears broken, unstyled, or missing.

**Solutions**:

1. Verify Badge component is imported correctly:

   ```tsx
   import { Badge } from '@grasdouble/lufa_design-system';
   ```

2. Ensure design system CSS is loaded in your application:

   ```tsx
   import '@grasdouble/lufa_design-system/dist/style.css';
   ```

3. Check that children prop is provided (required):

   ```tsx
   {
     /* ❌ Error - no children */
   }
   <Badge />;

   {
     /* ✅ Correct */
   }
   <Badge>Content</Badge>;
   ```

### Dot Indicator Not Showing

**Problem**: `dot={true}` is set but the dot doesn't appear.

**Solutions**:

1. Verify CSS classes are loaded (check browser DevTools):

   ```tsx
   // Should have these classes when dot={true}:
   // .badge-with-dot, .badge-dot, .badge-content
   <Badge dot>Check DevTools</Badge>
   ```

2. Ensure badge has content (dot requires content):

   ```tsx
   {
     /* ❌ Won't show dot properly */
   }
   <Badge dot></Badge>;

   {
     /* ✅ Correct */
   }
   <Badge dot>5</Badge>;
   ```

3. Check if custom CSS is overriding dot styles:
   ```css
   /* Avoid overriding these classes */
   .badge-dot {
     /* ... */
   }
   .badge-content {
     /* ... */
   }
   ```

### Badge Text Truncated or Wrapping

**Problem**: Badge text is cut off or wrapping to multiple lines.

**Solutions**:

1. Keep badge text short (1-3 words or numbers only):

   ```tsx
   {/* ❌ Too long */}
   <Badge>This is a very long badge text that wraps</Badge>

   {/* ✅ Concise */}
   <Badge>Active</Badge>
   <Badge>3 new</Badge>
   ```

2. Use Text component for longer content:
   ```tsx
   <Text>
     Status: <Badge>Active</Badge> (Updated 5 minutes ago)
   </Text>
   ```

### Badge Not Inline with Text

**Problem**: Badge appears on new line or doesn't align with surrounding text.

**Solutions**:

1. Ensure badge is used inline with text (default is `as="span"`):

   ```tsx
   <p>
     Status: <Badge variant="success">Active</Badge>
   </p>
   ```

2. Check parent element display properties:

   ```tsx
   {
     /* ✅ Correct - flex layout */
   }
   <div style={{ display: 'flex', alignItems: 'center', gap: '8px' }}>
     <span>User:</span>
     <Badge>Admin</Badge>
   </div>;
   ```

3. Use CSS flexbox for proper alignment:
   ```css
   .status-container {
     display: flex;
     align-items: center;
     gap: 8px;
   }
   ```

**Solutions**:

   ```tsx
   // Check in browser DevTools that CSS custom properties exist:
   // --lufa-component-badge-success-background
   ```

   ```tsx
   {
   ```

   ```css
   /* ❌ Don't override variant styles */

   ```

### TypeScript Errors with Polymorphic Props

**Problem**: TypeScript complains about props when using `as` prop.

**Solutions**:

1. Ensure props match the element type:

   ```tsx
   {
     /* ❌ Error - htmlFor only valid for label */
   }
   <Badge as="span" htmlFor="input">
     Error
   </Badge>;

   {
     /* ✅ Correct */
   }
   <Badge as="label" htmlFor="input">
     Label
   </Badge>;
   ```

2. Use type assertion if necessary (not recommended):

   ```tsx
   <Badge as="label" {...(labelProps as any)} />
   ```

3. Define proper types for wrapper components:

   ```tsx
   import type { ElementType } from 'react';

   import type { BadgeProps } from '@grasdouble/lufa_design-system';

   function Wrapper<T extends ElementType = 'span'>(props: BadgeProps<T>) {
     return <Badge {...props} />;
   }
   ```
