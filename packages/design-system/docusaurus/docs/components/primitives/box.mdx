# Box

<DarkModeCompatible />

A flexible, polymorphic container component that serves as the foundation for all layout compositions in the Lufa Design System.

## Overview

Use `Box` when you need…

- A universal layout primitive with utility-based spacing props (`padding`, `margin`)
- Semantic HTML elements via the polymorphic `as` prop (`section`, `article`, `header`, etc.)
- Quick prototyping with background colors, borders, and border radius
- A performance-optimized container (uses CSS classes, not inline styles)
- A foundation component for building complex layouts and compositions
- Token-based design values that automatically adapt to light/dark themes

## Basic Usage

```tsx title="src/App.tsx"
import { Box } from '@grasdouble/lufa_design-system';

function App() {
  return (
    <>
      {/* Simple container with padding */}
      <Box padding="default">Content</Box>

      {/* Card-like box */}
      <Box padding="comfortable" background="surface" borderRadius="medium" borderWidth="thin" borderColor="default">
        Card content
      </Box>

      {/* Semantic section */}
      <Box as="section" padding="spacious">
        <h2>Section Title</h2>
      </Box>
    </>
  );
}
```

## Anatomy

The Box component is a single container element with customizable spacing, background, and border properties:

```
┌───────────────────────────────────────────────────┐
│                   ↑ margin-top                    │
│   ┌───────────────────────────────────────────┐   │
│   │           ↑ padding-top                   │   │
│   │   ┌───────────────────────────────────┐   │   │
│ m │ p │                                   │ p │ m │
│ a │ a │          Content Area             │ a │ a │
│ r │ d │                                   │ d │ r │
│ g │ d │                                   │ d │ g │
│ i │ i │                                   │ i │ i │
│ n │ n │                                   │ n │ n │
│   │ g │                                   │ g │   │
│ l │   └───────────────────────────────────┘   │ r │
│ e │           ↓ padding-bottom                │ i │
│ f │                                           │ g │
│ t └───────────────────────────────────────────┘ h │
│                  ↓ margin-bottom                t │
└───────────────────────────────────────────────────┘
```

**Component Structure:**

- **Container**: Single element (default `div`, customizable via `as` prop)
- **Padding**: Inner spacing between border and content (supports all directions)
- **Margin**: Outer spacing between Box and adjacent elements (supports all directions)
- **Border**: Optional border with customizable width, color, and radius
- **Background**: Optional semantic background color from design tokens

**Visual Properties:**

- **Spacing**: Uses semantic tokens (`none`, `tight`, `compact`, `default`, `comfortable`, `spacious`)
- **Background**: Semantic color tokens (`page`, `surface`, `success`, `error`, `warning`, `info`, etc.)
- **Border Radius**: Rounded corners (`none`, `small`, `default`, `medium`, `large`, `full`)
- **Border Width**: Border thickness (`none`, `thin`, `medium`, `thick`)
- **Border Color**: Semantic border colors (`default`, `strong`, `success`, `error`, `warning`, `info`)
- **Display**: CSS display property (`block`, `inline-block`, `flex`, `inline-flex`, `grid`, `none`)

### Spacing Values (Padding & Margin)

All spacing props use the same semantic token scale:

- **none**: `0px` - No spacing (useful for overriding)
- **tight**: `4px` - Minimal spacing for dense layouts
- **compact**: `8px` - Small spacing for compact components
- **default**: `16px` - Standard spacing for most use cases (recommended default)
- **comfortable**: `24px` - Generous spacing for breathing room
- **spacious**: `32px` - Large spacing for major sections

**Directional Props:**

- `padding` / `margin`: Applies to all four sides equally
- `paddingX` / `marginX`: Horizontal only (left + right)
- `paddingY` / `marginY`: Vertical only (top + bottom)
- `paddingTop`, `paddingRight`, `paddingBottom`, `paddingLeft`: Individual sides
- `marginTop`, `marginRight`, `marginBottom`, `marginLeft`: Individual sides

**Prop Priority:** Individual sides override axis props, which override all-sides props.

```tsx
<Box
  padding="default" // 16px all sides
  paddingX="spacious" // Overrides left/right to 32px
  paddingTop="tight" // Overrides top to 4px
>
  Result: 4px top, 32px right, 16px bottom, 32px left
</Box>
```

### Background Values

Semantic background colors that automatically adapt to light/dark themes:

- **page**: Page background (lightest in light mode, darkest in dark mode)
- **surface**: Surface layer (cards, panels)
- **success**: Success state background (green tones)
- **error**: Error state background (red tones)
- **warning**: Warning state background (amber/yellow tones)
- **info**: Informational background (blue tones)
- **overlay**: Semi-transparent overlay (modals, dropdowns)
- **on-primary**: Background for content on primary brand color (AAA contrast)
- **on-secondary**: Background for content on secondary color (AAA contrast)
- **on-success**: Background for content on success color (AAA contrast)
- **on-error**: Background for content on error color (AAA contrast)
- **on-warning**: Background for content on warning color (AAA contrast)
- **on-info**: Background for content on info color (AAA contrast)

All background colors meet WCAG 2.1 AA contrast requirements for text readability.

### Border Values

**Border Radius:**

- **none**: `0px` - Sharp corners
- **small**: `4px` - Subtle rounding
- **default**: `8px` - Standard rounding (recommended)
- **medium**: `12px` - Moderate rounding
- **large**: `16px` - Prominent rounding
- **full**: `9999px` - Fully rounded (creates circles/pills)

**Border Width:**

- **none**: `0px` - No border
- **thin**: `1px` - Subtle border (recommended for most borders)
- **medium**: `2px` - Visible border
- **thick**: `4px` - Strong emphasis border

**Border Color:**

- **default**: Standard border color (subtle)
- **strong**: Emphasized border (higher contrast)
- **success**: Success state border (green)
- **error**: Error state border (red)
- **warning**: Warning state border (amber)
- **info**: Info state border (blue)

To create a visible border, both `borderWidth` and `borderColor` must be set.

### Display Values

Controls the CSS `display` property:

- **block**: Full width, stacks vertically (default for `div`)
- **inline-block**: Inline flow but respects width/height
- **flex**: Flexbox container for child layout
- **inline-flex**: Inline flexbox container
- **grid**: CSS Grid container for child layout
- **none**: Hides the element

Use `display="flex"` or `display="grid"` with additional `style` prop for layout properties:

```tsx
<Box display="flex" style={{ gap: '16px', flexDirection: 'column' }}>
  <Box>Item 1</Box>
  <Box>Item 2</Box>
</Box>
```

### Polymorphic Element (`as` prop)

Renders Box as different HTML elements for semantic markup:

- **div** (default): Generic container
- **section**: Thematic content section
- **article**: Self-contained composition
- **header**: Introductory content or navigation
- **footer**: Footer for nearest sectioning element
- **main**: Main content of the document
- **nav**: Navigation links section
- **aside**: Content tangentially related to main content

The `as` prop provides full type safety - props are typed based on the chosen element.

```tsx
// Type-safe: accepts <section> HTML attributes
<Box as="section" id="hero" aria-labelledby="hero-title">
  <h2 id="hero-title">Hero Section</h2>
</Box>
```

## Best Practices

### Do ✅

- Use semantic HTML via the `as` prop for better accessibility (`<Box as="section">` instead of `<div>`)
- Combine `borderWidth` and `borderColor` props together (both are required for visible borders)
- Use spacing tokens consistently across your app for visual harmony
- Leverage directional props (`paddingX`, `marginY`) for concise code
- Use `background="surface"` for card-like components that need elevation
- Prefer `Box` over custom divs for consistent spacing and token usage

### Don't ❌

- Don't use `Box` for everything - use semantic HTML directly when no utility props are needed
- Don't mix custom inline styles with utility props - prefer utility props for consistency
- Don't set `borderWidth` without `borderColor` (border won't be visible)
- Don't hardcode spacing values in `style` prop - use spacing props with semantic tokens
- Don't overuse deep nesting - keep component structure flat when possible
- Don't forget to set `as` prop for semantic landmarks (`main`, `nav`, `header`, `footer`)

## Accessibility

The Box component is designed with WCAG 2.1 AA compliance in mind and serves as an accessible layout primitive.

### Keyboard Navigation

Box is a **non-interactive container by default** and does not receive keyboard focus. However:

- **When used with `tabIndex={0}`**: Box becomes focusable and can be navigated via Tab key
- **When used with click handlers**: Box can be activated with Enter key if `tabIndex` is set
- **When polymorphic (`as`)**: Semantic elements like `<nav>` may have native keyboard behavior

```tsx
// Non-focusable container (default)
<Box padding="default">Content</Box>

// Focusable interactive box
<Box
  tabIndex={0}
  onClick={handleClick}
  onKeyDown={(e) => {
    if (e.key === 'Enter' || e.key === ' ') {
      handleClick();
    }
  }}
  padding="default"
>
  Interactive content
</Box>
```

### Screen Reader Support

**For Standard Container:**

```tsx
<Box padding="default">Regular content</Box>
// Screen reader announces: "content" (no specific role)
```

**For Semantic Elements:**

```tsx
<Box as="nav" aria-label="Main navigation">
  <ul>...</ul>
</Box>
// Screen reader announces: "Main navigation, navigation landmark"

<Box as="main">
  <h1>Page Title</h1>
</Box>
// Screen reader announces: "Page Title, main landmark"
```

**With Custom ARIA:**

```tsx
<Box role="region" aria-label="Product details">
  Product information
</Box>
// Screen reader announces: "Product details, region"
```

### Best Practices for Accessibility

#### Do ✅

- Use semantic HTML elements via `as` prop for landmarks (`<Box as="main">`, `<Box as="nav">`)
- Provide `aria-label` or `aria-labelledby` for landmark regions
- Ensure sufficient color contrast when using `background` prop (all token values meet WCAG AA)
- Use appropriate heading hierarchy within Box containers
- Add `tabIndex={0}` only when Box needs to be focusable (interactive content)
- Combine Box with proper heading structure for document outline

#### Don't ❌

- Don't use `<div>` when semantic HTML is appropriate (use `as="section"`, `as="nav"`, etc.)
- Don't create keyboard traps by making all containers focusable unnecessarily
- Don't rely solely on `background` color to convey meaning (combine with text/icons)
- Don't nest interactive Box elements inside each other (e.g., clickable Box inside clickable Box)
- Don't skip heading levels when structuring content inside Box
- Don't use `role="button"` on Box - use actual `<button>` elements instead

### ARIA Attributes

The Box component supports all standard ARIA attributes as props:

```tsx
<Box role="region" aria-label="Sidebar navigation" aria-describedby="sidebar-description" aria-live="polite">
  Dynamic content
</Box>
```

**Commonly used ARIA attributes:**

- `role`: Defines the role of the element (`region`, `group`, `banner`, etc.)
- `aria-label`: Accessible name for the region
- `aria-labelledby`: ID of element that labels this region
- `aria-describedby`: ID of element that describes this region
- `aria-live`: Announces dynamic content changes (`polite`, `assertive`, `off`)
- `aria-hidden`: Hides element from assistive technology

**Automatic ARIA handling:**

The Box component does not automatically add ARIA attributes. You must add them explicitly based on your use case.

### Color Contrast

All `background` prop values meet **WCAG 2.1 AA** contrast requirements:

- **Normal text**: Minimum 4.5:1 contrast ratio
- **Large text** (18pt+): Minimum 3:1 contrast ratio
- **UI components**: Minimum 3:1 contrast ratio against adjacent colors

**Tested in both light and dark modes**. All semantic background tokens automatically adapt to the active theme while maintaining contrast requirements.

**Examples:**

```tsx
{
  /* ✅ WCAG AA compliant: surface background + default text color */
}
<Box background="surface" padding="default">
  <p>Readable text content</p>
</Box>;

{
  /* ✅ WCAG AAA compliant: on-primary background guarantees high contrast */
}
<Box background="on-primary" padding="default">
  <p>High contrast text</p>
</Box>;

{
  /* ⚠️ Check contrast: custom text color may not meet WCAG */
}
<Box background="warning" padding="default">
  <p style={{ color: 'yellow' }}>May have insufficient contrast</p>
</Box>;
```

## Common Patterns

### Pattern 1: Card Component

Create a reusable card layout with Box as the foundation.

```tsx title="src/components/Card.tsx"
import type { ReactNode } from 'react';

import { Box } from '@grasdouble/lufa_design-system';

interface CardProps {
  children: ReactNode;
  title?: string;
  actions?: ReactNode;
}

export function Card({ children, title, actions }: CardProps) {
  return (
    <Box padding="comfortable" background="surface" borderRadius="medium" borderWidth="thin" borderColor="default">
      {title && (
        <Box as="header" marginBottom="default">
          <h3 style={{ margin: 0, fontSize: '18px', fontWeight: 600 }}>{title}</h3>
        </Box>
      )}

      <Box>{children}</Box>

      {actions && (
        <Box as="footer" marginTop="default" display="flex" style={{ gap: '8px' }}>
          {actions}
        </Box>
      )}
    </Box>
  );
}
```

### Pattern 2: Responsive Spacing with Composition

Combine Box components for complex layouts with consistent spacing.

```tsx title="src/components/PageLayout.tsx"
import { Box } from '@grasdouble/lufa_design-system';

export function PageLayout() {
  return (
    <Box as="main" padding="spacious">
      {/* Page header */}
      <Box as="header" marginBottom="comfortable">
        <h1>Page Title</h1>
        <p>Page description with proper spacing</p>
      </Box>

      {/* Content sections */}
      <Box as="section" marginBottom="spacious">
        <h2>Section 1</h2>
        <Box padding="default" background="surface" borderRadius="default">
          Section content with elevated surface
        </Box>
      </Box>

      <Box as="section" marginBottom="spacious">
        <h2>Section 2</h2>
        <Box padding="default" background="surface" borderRadius="default">
          Another section with consistent spacing
        </Box>
      </Box>

      {/* Page footer */}
      <Box as="footer" marginTop="spacious" paddingTop="default" borderWidth="thin" borderColor="default">
        <p>Footer content</p>
      </Box>
    </Box>
  );
}
```

### Pattern 4: Grid Layout for Cards

Use Box with `display="grid"` for responsive card grids.

```tsx title="src/components/CardGrid.tsx"
import { Box } from '@grasdouble/lufa_design-system';

export function CardGrid() {
  const items = [
    { id: 1, title: 'Item 1', description: 'Description 1' },
    { id: 2, title: 'Item 2', description: 'Description 2' },
    { id: 3, title: 'Item 3', description: 'Description 3' },
    { id: 4, title: 'Item 4', description: 'Description 4' },
  ];

  return (
    <Box
      display="grid"
      padding="spacious"
      style={{
        gridTemplateColumns: 'repeat(auto-fill, minmax(250px, 1fr))',
        gap: '24px',
      }}
    >
      {items.map((item) => (
        <Box
          key={item.id}
          padding="comfortable"
          background="surface"
          borderRadius="medium"
          borderWidth="thin"
          borderColor="default"
        >
          <h3>{item.title}</h3>
          <p>{item.description}</p>
        </Box>
      ))}
    </Box>
  );
}
```

## TypeScript Support

The Box component is fully typed with TypeScript and includes advanced polymorphic typing:

```tsx

/**
 * Spacing values based on semantic tokens
 */
type SpacingValue = 'none' | 'tight' | 'compact' | 'default' | 'comfortable' | 'spacious';

/**
 * Background color values based on semantic tokens
 */
type BackgroundValue =
  | 'page'
  | 'surface'
  | 'success'
  | 'error'
  | 'warning'
  | 'info'
  | 'overlay'
  | 'on-primary'
  | 'on-secondary'
  | 'on-success'
  | 'on-error'
  | 'on-warning'
  | 'on-info';

/**
 * Border radius values based on semantic tokens
 */
type BorderRadiusValue = 'none' | 'small' | 'default' | 'medium' | 'large' | 'full';

/**
 * Border width values
 */
type BorderWidthValue = 'none' | 'thin' | 'medium' | 'thick';

/**
 * Border color values based on semantic tokens
 */
type BorderColorValue = 'default' | 'strong' | 'success' | 'error' | 'warning' | 'info';

/**
 * Display values (CSS display property)
 */
type DisplayValue = 'block' | 'inline-block' | 'flex' | 'inline-flex' | 'grid' | 'none';

/**
 * Valid HTML elements for polymorphic rendering
 */
type ValidBoxElements = 'div' | 'section' | 'article' | 'header' | 'footer' | 'main' | 'nav' | 'aside';

/**
 * Box component props with polymorphic `as` prop support
 */
export interface BoxProps<T extends ElementType = 'div'> {
  as?: T;

  // Spacing - Padding
  padding?: SpacingValue;
  paddingX?: SpacingValue;
  paddingY?: SpacingValue;
  paddingTop?: SpacingValue;
  paddingRight?: SpacingValue;
  paddingBottom?: SpacingValue;
  paddingLeft?: SpacingValue;

  // Spacing - Margin
  margin?: SpacingValue;
  marginX?: SpacingValue;
  marginY?: SpacingValue;
  marginTop?: SpacingValue;
  marginRight?: SpacingValue;
  marginBottom?: SpacingValue;
  marginLeft?: SpacingValue;

  // Background
  background?: BackgroundValue;

  // Border
  borderRadius?: BorderRadiusValue;
  borderWidth?: BorderWidthValue;
  borderColor?: BorderColorValue;

  // Display
  display?: DisplayValue;

  // Standard props
  className?: string;
  children?: ReactNode;
}

/**
 * Combined props type including element-specific props

type BoxComponentProps<T extends ElementType> = BoxProps<T> & Omit<ComponentPropsWithoutRef<T>, keyof BoxProps<T>>;
```

**Benefits of TypeScript typing:**

- **Type safety**: Props are validated at compile time
- **IntelliSense**: Auto-complete for all prop values in your IDE
- **Polymorphic typing**: When using `as` prop, element-specific attributes are typed correctly
- **Documentation**: Hover over props to see JSDoc descriptions

**Example with type safety:**

```tsx
// ✅ Type-safe: 'section' allows 'id' attribute

// ✅ Type-safe: autocomplete shows SpacingValue options
```

## Troubleshooting

### Issue 1: Border is not visible

**Problem**: Set `borderWidth` prop but border doesn't appear.

**Solution**: Both `borderWidth` and `borderColor` must be set for a visible border.

```tsx
{
  /* ❌ Border not visible - missing borderColor */
}
<Box borderWidth="thin">Content</Box>;

{
  /* ✅ Border visible - both props set */
}
<Box borderWidth="thin" borderColor="default">
  Content
</Box>;
```

### Issue 2: Spacing props not working as expected

**Problem**: Setting multiple spacing props and one overrides the other unexpectedly.

**Solution**: Individual side props override axis props, which override all-sides props. Check the priority order.

```tsx
{
  /* Priority: paddingTop (4px) > paddingY > padding */
}
<Box
  padding="default" // 16px all sides
  paddingY="spacious" // Overrides top/bottom to 32px
  paddingTop="tight" // Overrides only top to 4px
>
  {/* Result: 4px top, 16px right, 32px bottom, 16px left */}
  Content
</Box>;
```

### Issue 3: TypeScript errors with `as` prop

**Problem**: TypeScript complains about props when using `as` prop with custom element.

**Solution**: Ensure you're using element-specific props correctly. The Box component automatically types props based on the `as` value.

```tsx
{
  /* ❌ Type error: 'href' is not valid for <section> */
}
<Box as="section" href="/page">
  Content
</Box>;

{
  /* ✅ Correct: 'href' only works with <a> (not supported in Box) */
}
<a href="/page">
  <Box padding="default">Content</Box>
</a>;
```

### Issue 4: Dark mode background looks incorrect

**Problem**: Background color doesn't adapt properly to dark mode.

**Solution**: Use semantic background tokens (e.g., `surface`, `page`) instead of hardcoded colors. Semantic tokens automatically adapt to the theme.

```tsx
{
  /* ❌ Hardcoded color doesn't adapt to dark mode */
}
<Box style={{ backgroundColor: '#ffffff' }}>Content</Box>;

{
  /* ✅ Semantic token adapts automatically */
}
<Box background="surface">Content</Box>;
```

### Issue 5: Nested Box with conflicting margins

**Problem**: Margin collapsing or unexpected spacing between nested Boxes.

**Solution**: CSS margin collapsing is standard behavior. Use `padding` on parent or `marginTop="none"` on first child to prevent collapsing.

```tsx
{
  /* Margin collapsing may occur */
}
<Box marginBottom="default">
  <Box marginTop="default">Child 1</Box>
  <Box marginTop="default">Child 2</Box>
</Box>;

{
  /* Solution: Use padding on parent or prevent collapsing */
}
<Box padding="default">
  <Box marginBottom="default">Child 1</Box>
  <Box>Child 2</Box>
</Box>;
```
