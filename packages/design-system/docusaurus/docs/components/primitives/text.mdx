# Text

<DarkModeCompatible />

A flexible, polymorphic typography component that provides consistent text styling across the Lufa Design System.

## Overview

Use `Text` when you need…

- Consistent typography using semantic type scales (h1-h6, body, caption, label)
- Semantic text colors that adapt to light/dark themes
- Font weight control (normal, medium, semibold, bold)
- Text alignment (left, center, right, justify)
- Text transformations (uppercase, lowercase, capitalize)
- Semantic HTML elements via the polymorphic `as` prop (headings, paragraphs, spans, labels)
- Performance-optimized typography (uses CSS classes, not inline styles)
- Token-based design values that automatically adapt to themes

```tsx title="src/App.tsx"
function App() {

      </Text>

      </Text>

      {/* Label with uppercase */}

      </Text>

      {/* Caption */}
      <Text variant="caption" color="tertiary" align="center">
        Image caption text
      </Text>
    </>
  );
}
```

## Anatomy

The Text component is a single typography element with customizable font properties:

```
┌──────────────────────────────────────────┐
│  Text Component                          │
│  ┌────────────────────────────────────┐  │
│  │                                    │  │
│  │  variant: h1-h6, body, caption    │  │
│  │  color: primary, secondary, etc.  │  │
│  │  weight: normal, medium, bold     │  │
│  │  align: left, center, right       │  │
│  │  transform: uppercase, etc.       │  │
│  │                                    │  │
│  │  The quick brown fox jumps...     │  │
│  │                                    │  │
│  └────────────────────────────────────┘  │
└──────────────────────────────────────────┘
```

**Component Structure:**

- **Container**: Single text element (default `p`, customizable via `as` prop)
- **Variant**: Typography scale from design tokens (h1-h6, body variants, caption, label)
- **Color**: Semantic text color tokens (primary, secondary, success, error, etc.)
- **Weight**: Font weight values (normal 400, medium 500, semibold 600, bold 700)
- **Align**: Text alignment property (left, center, right, justify)
- **Transform**: Text transformation (none, uppercase, lowercase, capitalize)

**Typography Properties:**

- **Variants**: Uses semantic typography tokens for consistent sizing and line-height
- **Colors**: Semantic color tokens that automatically adapt to light/dark mode
- **Weights**: Standard font-weight values for emphasis hierarchy
- **Responsive**: Typography scales adapt appropriately across breakpoints

## API Reference

### Props

| Prop        | Type                                                                                                             | Default     | Description                            |
| ----------- | ---------------------------------------------------------------------------------------------------------------- | ----------- | -------------------------------------- |
| `as`        | `'p' \| 'span' \| 'div' \| 'h1' \| 'h2' \| 'h3' \| 'h4' \| 'h5' \| 'h6' \| 'label' \| 'legend' \| 'figcaption'`  | `'p'`       | HTML element to render                 |
| `variant`   | `'h1' \| 'h2' \| 'h3' \| 'h4' \| 'h5' \| 'h6' \| 'body-large' \| 'body' \| 'body-small' \| 'caption' \| 'label'` | `'body'`    | Typography variant (semantic scale)    |
| `color`     | `'primary' \| 'secondary' \| 'tertiary' \| 'success' \| 'error' \| 'warning' \| 'info' \| 'inverse'`             | `'primary'` | Text color using semantic tokens       |
| `weight`    | `'normal' \| 'medium' \| 'semibold' \| 'bold'`                                                                   | `'normal'`  | Font weight                            |
| `align`     | `'left' \| 'center' \| 'right' \| 'justify'`                                                                     | `'left'`    | Text alignment                         |
| `transform` | `'none' \| 'uppercase' \| 'lowercase' \| 'capitalize'`                                                           | `'none'`    | Text transformation                    |
| `className` | `string`                                                                                                         | `undefined` | Additional CSS classes                 |
| `children`  | `ReactNode`                                                                                                      | `undefined` | Text content to render inside the Text |

Also supports all standard HTML attributes for the underlying element (e.g., `id`, `role`, `aria-*`, `data-*`, `style`, event handlers like `onClick`, `onKeyDown`, etc.).

### Variant Values

Typography scale values based on semantic tokens:

- **h1**: Largest heading (`--lufa-semantic-typography-heading-1`) - ~40px
- **h2**: Large heading (`--lufa-semantic-typography-heading-2`) - ~32px
- **h3**: Medium heading (`--lufa-semantic-typography-heading-3`) - ~28px
- **h4**: Small heading (`--lufa-semantic-typography-heading-4`) - ~24px
- **h5**: Extra small heading (`--lufa-semantic-typography-heading-5`) - ~20px
- **h6**: Smallest heading (`--lufa-semantic-typography-heading-6`) - ~18px
- **body-large**: Large body text (`--lufa-semantic-typography-body-large`) - ~18px
- **body**: Standard body text (`--lufa-semantic-typography-body`) - ~16px (recommended default)
- **body-small**: Small body text (`--lufa-semantic-typography-body-small`) - ~14px
- **caption**: Caption text (`--lufa-semantic-typography-caption`) - ~12px
- **label**: Label text (`--lufa-semantic-typography-label`) - ~14px

**Usage Guidelines:**

- Use heading variants (h1-h6) for section titles and hierarchy
- Use body variants for paragraphs and content text
- Use caption for supplementary information (image captions, footnotes)
- Use label for form field labels and metadata

```tsx
<Text variant="h1">Main Page Title</Text>
<Text variant="h2">Section Title</Text>
<Text variant="body">Paragraph content</Text>
<Text variant="caption">Image caption or footnote</Text>
<Text variant="label">Form field label</Text>
```

### Color Values

Semantic text colors that automatically adapt to light/dark themes:

- **primary**: Default text color (highest contrast, most readable)
- **secondary**: Secondary text color (medium emphasis)
- **tertiary**: Tertiary text color (least emphasis, subtle)
- **success**: Success state text (green tones)
- **error**: Error state text (red tones)
- **warning**: Warning state text (amber/yellow tones)
- **info**: Informational text (blue tones)
- **inverse**: Inverse text color (for use on dark backgrounds in light mode, light backgrounds in dark mode)

All text colors meet **WCAG 2.1 AA** contrast requirements for normal text (4.5:1) and large text (3:1).

```tsx
<Text color="primary">Default text</Text>
<Text color="secondary">Less emphasized text</Text>
<Text color="success">✓ Success message</Text>
<Text color="error">✗ Error message</Text>
<Text color="inverse" style={{ background: 'var(--lufa-token-color-background-primary)' }}>
  Text on colored background
</Text>
```

### Weight Values

Font weight values for emphasis and hierarchy:

- **normal**: Regular weight (400) - Default, most readable for body text
- **medium**: Medium weight (500) - Slight emphasis
- **semibold**: Semi-bold weight (600) - Strong emphasis, good for labels
- **bold**: Bold weight (700) - Maximum emphasis, good for headings

```tsx
<Text weight="normal">Regular text</Text>
<Text weight="medium">Medium emphasis</Text>
<Text weight="semibold">Strong emphasis</Text>
<Text weight="bold">Bold emphasis</Text>
```

### Align Values

Text alignment property:

- **left**: Left-aligned (default for LTR languages)
- **center**: Center-aligned (good for headings, captions)
- **right**: Right-aligned (good for numbers, timestamps)
- **justify**: Justified text (full width alignment)

```tsx
<Text align="left">Left-aligned text (default)</Text>
<Text align="center">Center-aligned text</Text>
<Text align="right">Right-aligned text</Text>
<Text align="justify">
  Justified text that fills the full width of the container with equal spacing.
</Text>
```

### Transform Values

Text transformation property:

- **none**: No transformation (default, preserves original casing)
- **uppercase**: All uppercase letters (good for labels, badges)
- **lowercase**: All lowercase letters
- **capitalize**: Capitalize first letter of each word

```tsx
<Text transform="none">Original Casing</Text>
<Text transform="uppercase">all uppercase</Text>
<Text transform="lowercase">ALL LOWERCASE</Text>
<Text transform="capitalize">capitalize each word</Text>
```

### Polymorphic Element (`as` prop)

Renders Text as different HTML elements for semantic markup:

- **p** (default): Paragraph element for body text
- **span**: Inline text element (no line break)
- **div**: Block-level generic container
- **h1, h2, h3, h4, h5, h6**: Heading elements for document structure
- **label**: Form label element (associates with form controls)
- **legend**: Fieldset legend element
- **figcaption**: Figure caption element

The `as` prop provides full type safety - props are typed based on the chosen element.

```tsx
// Semantic headings
<Text as="h1" variant="h1">Page Title</Text>
<Text as="h2" variant="h2">Section Title</Text>

// Inline text
<Text as="span" variant="body">Inline text</Text>

// Form labels
<Text as="label" variant="label" htmlFor="email">Email Address</Text>

// Figure captions
<Text as="figcaption" variant="caption" align="center">
  Figure 1: Chart description
</Text>
```

## Best Practices

### Do ✅

- Use semantic HTML via the `as` prop for better accessibility (`<Text as="h1">` instead of `<Text variant="h1">` with default p)
- Match variant to semantic element (use `variant="h1"` with `as="h1"`)
- Use semantic colors for state-based messaging (success, error, warning, info)
- Prefer semantic color tokens over custom text colors for theme consistency
- Use weight prop for emphasis hierarchy (normal → medium → semibold → bold)
- Use transform="uppercase" sparingly (labels, small UI elements only)
- Combine Text with layout components (Stack, Box) for proper spacing

### Don't ❌

- Don't use heading variants (h1-h6) without semantic heading elements
- Don't use transform="uppercase" for long passages of text (reduces readability)
- Don't hard-code text colors in style prop - use color prop with semantic tokens
- Don't skip heading levels in hierarchy (h1 → h2 → h3, not h1 → h3)
- Don't use Text for everything - use plain HTML text when no styling is needed
- Don't use justify alignment for short lines (causes awkward spacing)
- Don't mix too many weights in close proximity (creates visual noise)

## Accessibility

The Text component is designed with WCAG 2.1 AA compliance in mind and serves as an accessible typography primitive.

### Semantic HTML

**For Headings:**

Always use semantic heading elements with corresponding variants:

```tsx
// ✅ Good: Semantic h1 with h1 variant
<Text as="h1" variant="h1">Page Title</Text>

// ❌ Bad: Paragraph with h1 variant (no semantic meaning)
<Text variant="h1">Page Title</Text>

// ✅ Good: Complete heading hierarchy
<Text as="h1" variant="h1">Main Title</Text>
<Text as="h2" variant="h2">Section Title</Text>
<Text as="h3" variant="h3">Subsection Title</Text>
```

**For Body Text:**

```tsx
// ✅ Good: Default paragraph
<Text variant="body">Paragraph content</Text>

// ✅ Good: Inline text within paragraph
<Text variant="body">
  This is a paragraph with <Text as="span" weight="bold">bold text</Text> inline.
</Text>
```

**For Form Labels:**

```tsx
// ✅ Good: Semantic label with htmlFor
<Text as="label" variant="label" htmlFor="email-input">
  Email Address
</Text>
<input type="email" id="email-input" />

// ❌ Bad: Not a label element (no association)
<Text variant="label">Email Address</Text>
<input type="email" />
```

### Keyboard Navigation

Text is a **non-interactive element by default** and does not receive keyboard focus. However:

- **When used with `tabIndex={0}`**: Text becomes focusable and can be navigated via Tab key
- **For interactive text**: Consider using `<button>` or `<a>` elements instead

```tsx
// Non-focusable text (default)
<Text variant="body">Static text content</Text>

// Focusable text (for special cases like skip links)
<Text
  as="a"
  href="#main-content"
  tabIndex={0}
  variant="body"
>
  Skip to main content
</Text>
```

### Screen Reader Support

**For Standard Text:**

```tsx
<Text variant="body">Regular paragraph text</Text>
// Screen reader announces: "Regular paragraph text"
```

**For Headings:**

```tsx
<Text as="h1" variant="h1">Page Title</Text>
// Screen reader announces: "Page Title, heading level 1"

<Text as="h2" variant="h2">Section Title</Text>
// Screen reader announces: "Section Title, heading level 2"
```

**For Labels:**

```tsx
<Text as="label" htmlFor="username">Username</Text>
<input type="text" id="username" />
// Screen reader announces: "Username, edit text" (when focused on input)
```

**With Custom ARIA:**

```tsx
<Text role="status" aria-live="polite" color="success">
  ✓ Changes saved successfully
</Text>
// Screen reader announces: "Changes saved successfully" (politely)
```

### Best Practices for Accessibility

#### Do ✅

- Use semantic heading elements (h1-h6) with proper hierarchy
- Provide `aria-label` or `aria-labelledby` for custom roles
- Ensure sufficient color contrast (all semantic colors meet WCAG AA)
- Use appropriate heading levels (don't skip levels)
- Associate labels with form controls using `htmlFor`
- Use live regions (`aria-live`) for dynamic status messages
- Combine Text with semantic landmarks (within `<main>`, `<nav>`, `<aside>`)

#### Don't ❌

- Don't use `<div>` or `<p>` when heading elements are appropriate
- Don't skip heading levels (h1 → h3 without h2)
- Don't rely solely on color to convey meaning (combine with text/icons)
- Don't create overly complex heading hierarchies (too many levels)
- Don't use `aria-label` on non-interactive text without good reason
- Don't make text interactive without proper keyboard support

### ARIA Attributes

The Text component supports all standard ARIA attributes as props:

```tsx
<Text role="heading" aria-level={2} aria-labelledby="section-title">
  Dynamic heading
</Text>

<Text role="status" aria-live="assertive" color="error">
  ✗ Error: Please correct the form fields below
</Text>

<Text as="label" aria-describedby="email-help" htmlFor="email">
  Email Address
</Text>
<Text id="email-help" variant="caption" color="secondary">
  We'll never share your email
</Text>
```

**Commonly used ARIA attributes:**

- `role`: Defines the role of the element (`heading`, `status`, `alert`, etc.)
- `aria-level`: Heading level for custom headings (1-6)
- `aria-label`: Accessible name for the element
- `aria-labelledby`: ID of element that labels this element
- `aria-describedby`: ID of element that describes this element
- `aria-live`: Announces dynamic content changes (`polite`, `assertive`, `off`)
- `aria-hidden`: Hides element from assistive technology

### Color Contrast

All `color` prop values meet **WCAG 2.1 AA** contrast requirements:

- **Normal text** (< 18pt or < 14pt bold): Minimum 4.5:1 contrast ratio
- **Large text** (≥ 18pt or ≥ 14pt bold): Minimum 3:1 contrast ratio

**Tested in both light and dark modes**. All semantic color tokens automatically adapt to the active theme while maintaining contrast requirements.

**Examples:**

```tsx
{
  /* ✅ WCAG AA compliant: primary color + default background */
}
<Text color="primary">High contrast text</Text>;

{
  /* ✅ WCAG AA compliant: secondary color (less emphasis but still readable) */
}
<Text color="secondary">Secondary text with sufficient contrast</Text>;

{
  /* ✅ WCAG AA compliant: inverse color on colored background */
}
<div style={{ background: 'var(--lufa-token-color-background-primary)' }}>
  <Text color="inverse">Text on colored background</Text>
</div>;

{
  /* ⚠️ Check contrast: custom colors may not meet WCAG */
}
<Text style={{ color: '#888888' }}>May have insufficient contrast depending on background</Text>;
```

### Pattern 3: Form Field with Label and Description

Create accessible form fields with labels and helper text.

```tsx title="src/components/FormField.tsx"
import { Text } from '@grasdouble/lufa_design-system';

interface FormFieldProps {
  id: string;
  label: string;
  description?: string;
  required?: boolean;
  error?: string;
  children: React.ReactNode;
}

export function FormField({ id, label, description, required, error, children }: FormFieldProps) {
  const descriptionId = description ? `${id}-description` : undefined;
  const errorId = error ? `${id}-error` : undefined;

  return (
    <div style={{ marginBottom: '20px' }}>
      <Text as="label" variant="label" weight="semibold" htmlFor={id} style={{ display: 'block', marginBottom: '4px' }}>
        {label}
        {required && (
          <Text as="span" color="error">
            {' '}
            *
          </Text>
        )}
      </Text>

      {description && (
        <Text id={descriptionId} variant="caption" color="tertiary" style={{ display: 'block', marginBottom: '8px' }}>
          {description}
        </Text>
      )}

      {children}

      {error && (
        <Text id={errorId} variant="caption" color="error" role="alert" style={{ display: 'block', marginTop: '4px' }}>
          {error}
        </Text>
      )}
    </div>
  );
}

// Usage
<FormField
  id="email"
  label="Email Address"
  description="We'll never share your email with anyone else."
  required
  error="Please enter a valid email address"
>
  <input type="email" id="email" />
</FormField>;
```

## TypeScript Support

The Text component is fully typed with TypeScript and includes advanced polymorphic typing:

```tsx
import type { ComponentPropsWithoutRef, ElementType, ReactNode } from 'react';

/**
 * Typography variant values (semantic scale)
 */
type VariantValue =
  | 'h1'
  | 'h2'
  | 'h3'
  | 'h4'
  | 'h5'
  | 'h6'
  | 'body-large'
  | 'body'
  | 'body-small'
  | 'caption'
  | 'label';

/**
 * Text color values based on semantic tokens
 */
type ColorValue = 'primary' | 'secondary' | 'tertiary' | 'success' | 'error' | 'warning' | 'info' | 'inverse';

/**
 * Font weight values
 */
type WeightValue = 'normal' | 'medium' | 'semibold' | 'bold';

/**
 * Text alignment values
 */
type AlignValue = 'left' | 'center' | 'right' | 'justify';

/**
 * Text transformation values
 */
type TransformValue = 'none' | 'uppercase' | 'lowercase' | 'capitalize';

/**
 * Valid HTML elements for polymorphic rendering
 */
type ValidTextElements =
  | 'p'
  | 'span'
  | 'div'
  | 'h1'
  | 'h2'
  | 'h3'
  | 'h4'
  | 'h5'
  | 'h6'
  | 'label'
  | 'legend'
  | 'figcaption';

/**
 * Text component props with polymorphic `as` prop support
 */
export interface TextProps<T extends ElementType = 'p'> {
  as?: T;
  variant?: VariantValue;
  color?: ColorValue;
  weight?: WeightValue;
  align?: AlignValue;
  transform?: TransformValue;
  className?: string;
  children?: ReactNode;
}

/**
 * Combined props type including element-specific props
 */
type TextComponentProps<T extends ElementType> = TextProps<T> & Omit<ComponentPropsWithoutRef<T>, keyof TextProps<T>>;
```

**Benefits of TypeScript typing:**

- **Type safety**: Props are validated at compile time
- **IntelliSense**: Auto-complete for all prop values in your IDE
- **Polymorphic typing**: When using `as` prop, element-specific attributes are typed correctly
- **Documentation**: Hover over props to see JSDoc descriptions

**Example with type safety:**

```tsx
// ✅ Type-safe: 'h1' allows heading-specific attributes
<Text as="h1" variant="h1" id="main-title">Title</Text>

// ✅ Type-safe: autocomplete shows VariantValue options
<Text variant="body">Content</Text>

// ❌ Type error: 'xxl' is not a valid VariantValue
<Text variant="xxl">Content</Text>

// ✅ Type-safe: label-specific attributes
<Text as="label" htmlFor="input-field" variant="label">
  Label Text
</Text>

// ❌ Type error: 'href' not valid for <label>
<Text as="label" href="/page">Text</Text>
```

## Troubleshooting

### Issue 1: Text color not visible

**Problem**: Set a text color but it doesn't appear or has poor contrast.

**Solution**: Ensure you're using semantic color tokens that adapt to the theme. Check the background color for sufficient contrast.

```tsx
{
  /* ❌ Custom color may not have sufficient contrast */
}
<Text style={{ color: '#888888' }}>Low contrast text</Text>;

{
  /* ✅ Semantic color with guaranteed contrast */
}
<Text color="secondary">Proper contrast text</Text>;
```

### Issue 2: Heading hierarchy issues

**Problem**: Screen readers announce incorrect heading levels or skip levels.

**Solution**: Always use semantic heading elements with matching variants and maintain proper hierarchy.

```tsx
{/* ❌ Bad: Visual h1 without semantic meaning */}
<Text variant="h1">Title</Text>

{/* ❌ Bad: Skipping heading level */}
<Text as="h1" variant="h1">Title</Text>
<Text as="h3" variant="h3">Skipped h2</Text>

{/* ✅ Good: Semantic headings with proper hierarchy */}
<Text as="h1" variant="h1">Main Title</Text>
<Text as="h2" variant="h2">Section Title</Text>
<Text as="h3" variant="h3">Subsection Title</Text>
```

### Issue 3: Text not aligning properly

**Problem**: Text alignment doesn't appear to work.

**Solution**: Text alignment requires the parent container to have defined width. Inline elements may not show alignment.

```tsx
{
  /* ❌ Alignment not visible on inline span */
}
<Text as="span" align="center">
  Inline text
</Text>;

{
  /* ✅ Alignment visible on block element or within container */
}
<Text align="center" style={{ display: 'block' }}>
  Centered text
</Text>;

{
  /* ✅ Or use block-level element */
}
<Text as="div" align="center">
  Centered text
</Text>;
```

### Issue 4: Transform not working as expected

**Problem**: Text transformation doesn't display correctly.

**Solution**: Text transformation is a CSS property and depends on the original text content. It doesn't change the actual data.

```tsx
{
  /* Text transform is visual only - data remains unchanged */
}
<Text transform="uppercase">hello world</Text>;
// Displays: HELLO WORLD
// But value remains: "hello world"

{
  /* For data transformation, modify the string directly */
}
<Text>{text.toUpperCase()}</Text>;
```

### Issue 5: Dark mode text not readable

**Problem**: Text color doesn't adapt properly to dark mode.

**Solution**: Use semantic color tokens instead of hardcoded colors. Semantic tokens automatically adapt to the theme.

```tsx
{
  /* ❌ Hardcoded color doesn't adapt to dark mode */
}
<Text style={{ color: '#000000' }}>Black text (not readable in dark mode)</Text>;

{
  /* ✅ Semantic color adapts automatically */
}
<Text color="primary">Adaptive text</Text>;
```

### Issue 6: Form label not associated with input

**Problem**: Clicking label doesn't focus the input field.

**Solution**: Use the `htmlFor` prop with the input's `id` to create the association.

```tsx
{/* ❌ Label not associated */}
<Text as="label">Email</Text>
<input type="email" id="email" />

{/* ✅ Label properly associated */}
<Text as="label" htmlFor="email">Email</Text>
<input type="email" id="email" />
```
