# Input

<DarkModeCompatible />

A versatile text input component for capturing user data.

## Overview

Use `Input` when you need…

- To capture short text data from the user (email, name, phone)
- To create form fields with consistent styling
- To support different interaction states (focus, error, disabled)

## Import

```tsx
import { Input } from '@grasdouble/lufa_design-system';
```

## Basic Usage

```tsx title="src/App.tsx"
import { Input, Label } from '@grasdouble/lufa_design-system';

function App() {
  return (
    <>
      <Label htmlFor="username">Username</Label>
      <Input id="username" placeholder="Enter username" />

      {/* With Error */}
      <Label htmlFor="email">Email</Label>
      <Input id="email" error placeholder="Enter email" />
    </>
  );
}
```

## API Reference

### Props

| Prop        | Type                  | Default     | Description                                                      |
| ----------- | --------------------- | ----------- | ---------------------------------------------------------------- |
| `error`     | `boolean`             | `false`     | Applies error styling (red border)                               |
| `disabled`  | `boolean`             | `false`     | Disables interaction and applies gray styling                    |
| `fullWidth` | `boolean`             | `false`     | Stretches the input to 100% of container width                   |
| `className` | `string`              | `undefined` | Additional CSS classes                                           |
| `...props`  | `InputHTMLAttributes` | -           | All standard HTML input attributes (type, value, onChange, etc.) |

## Best Practices

### Do ✅

- Always use with a `<Label>` component for accessibility.
- Use the `placeholder` attribute to provide a hint, not as a replacement for a label.
- Use `fullWidth` in mobile-first layouts.

### Don't ❌

- Don't use `Input` for multi-line text (use a Textarea component, to be implemented).
- Don't rely solely on color (`error` prop) to communicate failure; show an error message text as well.
