# Icon

<DarkModeCompatible />

A flexible icon component that provides uniform SVG rendering with semantic sizing and coloring based on design tokens.

## Overview

Use `Icon` when you need…

- Consistent icon rendering with semantic sizing (xs, sm, md, lg, xl)
- Semantic colors that adapt to light/dark themes
- Lucide React icon library integration with string-based API
- Accessibility support (title for screen readers, aria-hidden for decorative)
- Polymorphic rendering via `as` prop
- Performance-optimized rendering (CSS classes, not inline styles)
- Token-based design values that automatically adapt to themes

## Anatomy

The Icon component is a container element with an SVG icon inside:

```
┌────────────────────────────────────────────┐
│  Icon Component (Container)                │
│  ┌──────────────────────────────────────┐  │
│  │  as: element type (span/div/i)       │  │
│  │  size: dimensions (16px-40px)        │  │
│  │  color: semantic color token         │  │
│  │                                      │  │
│  │  ┌────────────────────────────────┐ │  │
│  │  │        SVG Icon                │ │  │
│  │  │    (Lucide React component)    │ │  │
│  │  └────────────────────────────────┘ │  │
│  └──────────────────────────────────────┘  │
└────────────────────────────────────────────┘
```

**Component Structure:**

- **Container**: Inline-flex wrapper element (default `span`, customizable via `as` prop)
- **Size**: Fixed dimensions using design tokens (xs: 16px, sm: 20px, md: 24px, lg: 32px, xl: 40px)
- **Color**: Semantic text color tokens (inherits or explicit colors)
- **SVG**: Lucide React icon component rendered inside container

**Accessibility Structure:**

- **Decorative** (no title): `aria-hidden="true"` on container
- **Accessible** (with title): `role="img"` + `aria-label` on container, visually hidden text span

## API Reference

### Props

| Prop | Type | Default | Description |
| ---- | ---- | ------- | ----------- |
| `name` | `IconName` | Required | Icon name from Lucide React library |
| `as` | `'span' \| 'div' \| 'i'` | `'span'` | HTML element to render as container |
| `size` | `'xs' \| 'sm' \| 'md' \| 'lg' \| 'xl'` | `'md'` | Size variant (dimensions) |
| `color` | `'currentColor' \| 'primary' \| 'secondary' \| 'success' \| 'error' \| 'warning' \| 'info' \| 'muted'` | `'currentColor'` | Color variant using semantic tokens |
| `title` | `string` | `undefined` | Accessible title for screen readers (makes icon non-decorative) |
| `className` | `string` | `undefined` | Additional CSS classes |

Also supports all standard HTML attributes for the underlying container element (e.g., `id`, `role`, `aria-*`, `data-*`, `style`, event handlers like `onClick`, `onKeyDown`, etc.).

### Icon Names

Available icon names from Lucide React library:

**User & Navigation:**
- `user`, `home`, `settings`, `menu`, `search`

**Actions:**
- `check`, `x`, `plus`, `minus`, `edit`, `trash`, `save`, `download`, `upload`

**Chevrons:**
- `chevron-down`, `chevron-up`, `chevron-left`, `chevron-right`

**Arrows:**
- `arrow-left`, `arrow-right`

**Status:**
- `alert-circle`, `info`, `check-circle`, `x-circle`

**Miscellaneous:**
- `external-link`, `eye`, `eye-off`, `heart`, `star`

More icons can be added to the icon map as needed. See [Lucide Icons](https://lucide.dev/icons) for the full library.

```tsx
<Icon name="user" />
<Icon name="check-circle" />
<Icon name="arrow-right" />
```

### Size Values

Icon size determines the width and height dimensions:

- **xs**: 16x16px - Smallest size, inline with small text
- **sm**: 20x20px - Small size, inline with body text  
- **md**: 24x24px - Default size, balanced for most uses (recommended)
- **lg**: 32x32px - Large size, prominent display
- **xl**: 40x40px - Extra large size, hero sections

```tsx
<Icon name="star" size="xs" />
<Icon name="star" size="sm" />
<Icon name="star" size="md" />
<Icon name="star" size="lg" />
<Icon name="star" size="xl" />
```

### Color Values

Semantic icon colors that automatically adapt to light/dark themes:

- **currentColor**: Inherits from parent text color (default, recommended for inline icons)
- **primary**: Default text color (highest contrast)
- **secondary**: Secondary text color (medium emphasis)
- **success**: Success state (green tones)
- **error**: Error state (red tones)
- **warning**: Warning state (amber tones)
- **info**: Informational (blue tones)
- **muted**: Muted/tertiary color (least emphasis)

All colors meet **WCAG 2.1 AA** contrast requirements.

```tsx
<Icon name="heart" color="currentColor" />
<Icon name="check-circle" color="success" />
<Icon name="x-circle" color="error" />
<Icon name="alert-circle" color="warning" />
<Icon name="info" color="info" />
```

### Title Prop (Accessibility)

The `title` prop controls whether the icon is decorative or accessible:

**Without title (decorative icon):**
- Adds `aria-hidden="true"` to hide from screen readers
- Use when icon is purely decorative or has adjacent text label

**With title (accessible icon):**
- Adds `role="img"` and `aria-label` for screen readers
- Renders visually hidden text span
- Use for icon-only buttons or standalone meaningful icons

```tsx
{/* Decorative - icon has adjacent text */}
<button>
  <Icon name="plus" size="sm" />
  <span>Add Item</span>
</button>

{/* Accessible - icon-only button */}
<button>
  <Icon name="trash" size="sm" color="error" title="Delete item" />
</button>
```

### Polymorphic Element (`as` prop)

Renders Icon container as different HTML elements:

- **span** (default): Inline element, best for inline usage
- **div**: Block-level element
- **i**: Inline element (semantic icon element in some systems)

The `as` prop provides full type safety - props are typed based on the chosen element.

```tsx
{/* Default span */}
<Icon name="star" />

{/* As div */}
<Icon as="div" name="star" />

{/* As i */}
<Icon as="i" name="star" />
```

## Best Practices

### Do ✅

- Use `currentColor` for icons that should inherit parent text color
- Always provide `title` for icon-only buttons or standalone meaningful icons
- Use semantic colors for status icons (success, error, warning, info)
- Match icon size to adjacent text size (sm with body-small, md with body, lg with headings)
- Use decorative icons (no title) when icon has adjacent text label
- Keep icon usage consistent across similar actions

### Don't ❌

- Don't use icon-only buttons without `title` prop
- Don't hard-code icon colors with style prop - use `color` prop with semantic tokens
- Don't use xl size excessively - it's for hero/focal elements only
- Don't mix multiple icon libraries in same project
- Don't override SVG styling directly - use component props
- Don't use icons as primary way to convey critical information without text alternatives

## Accessibility

The Icon component is designed with WCAG 2.1 AA compliance and provides flexible accessibility patterns.

### Decorative Icons

**When to use:** Icon is purely decorative or has adjacent text label

```tsx
{/* ✅ Good: Icon with adjacent text */}
<button>
  <Icon name="download" size="sm" />
  <span>Download File</span>
</button>

{/* ✅ Good: Icon in navigation with text */}
<nav>
  <a href="/home">
    <Icon name="home" size="sm" />
    <span>Home</span>
  </a>
</nav>
```

**Accessibility attributes:**
- `aria-hidden="true"` - Hides icon from screen readers
- No `role` or `aria-label` - Not announced to assistive technology

### Accessible Icons

**When to use:** Icon-only buttons, standalone meaningful icons, or icons that convey information

```tsx
{/* ✅ Good: Icon-only button with title */}
<button>
  <Icon name="settings" size="md" title="Open settings" />
</button>

{/* ✅ Good: Status icon with title */}
<Icon name="check-circle" size="lg" color="success" title="Operation successful" />

{/* ✅ Good: Navigation icon-only button */}
<button>
  <Icon name="menu" size="md" title="Open menu" />
</button>
```

**Accessibility attributes:**
- `role="img"` - Identifies as image for assistive technology
- `aria-label="{title}"` - Provides accessible name
- Visually hidden `<span>` with title text - Backup for older assistive technology

### Screen Reader Announcements

**Decorative icon:**
```tsx
<Icon name="chevron-right" />
// Screen reader: (skips, not announced)
```

**Accessible icon:**
```tsx
<Icon name="check" title="Completed" />
// Screen reader: "Completed, image"
```

**Icon in button:**
```tsx
<button>
  <Icon name="trash" title="Delete" />
</button>
// Screen reader: "Delete, button" (when focused)
```

### Keyboard Navigation

Icons are **non-interactive by default**:

- **Decorative icons**: Not focusable (not needed, adjacent element handles interaction)
- **Icon-only buttons**: Button receives focus, icon provides visual + accessible name

```tsx
{/* Icon not focusable - button is */}
<button>
  <Icon name="edit" title="Edit" />
</button>

{/* If icon must be focusable (rare), use tabIndex */}
<Icon name="info" title="Help information" tabIndex={0} role="img" />
```

### Best Practices for Accessibility

#### Do ✅

- Provide `title` for all icon-only interactive elements
- Use decorative icons (no title) when icon has adjacent text
- Ensure sufficient color contrast (all semantic colors meet WCAG AA)
- Test with screen readers (NVDA, JAWS, VoiceOver)
- Use semantic colors that convey meaning beyond color alone (e.g., success with check icon)
- Provide keyboard navigation for interactive icons

#### Don't ❌

- Don't omit `title` from icon-only buttons
- Don't rely solely on color to convey meaning (combine with icon shape and text)
- Don't make icons focusable unless they're standalone interactive elements
- Don't use `aria-label` on decorative icons
- Don't create overly complex icon-based interfaces without text alternatives

### ARIA Attributes

The Icon component supports all standard ARIA attributes:

```tsx
<Icon name="info" aria-describedby="help-text" title="Information" />

<Icon name="alert-circle" role="img" aria-label="Warning" color="warning" />

<button aria-label="Delete item">
  <Icon name="trash" size="sm" />
</button>
```

### Color Contrast

All `color` prop values meet **WCAG 2.1 AA** contrast requirements (4.5:1 for normal icons, 3:1 for large icons).

**Tested in both light and dark modes** - all semantic color tokens automatically adapt.

```tsx
{/* ✅ WCAG AA compliant */}
<Icon name="check" color="success" />
<Icon name="x" color="error" />

{/* ⚠️ Check contrast: custom colors may not meet WCAG */}
<Icon name="star" style={{ color: '#888888' }} />
```

### Pattern 2: Status Message with Icon

Display status messages with appropriate icons and colors.

```tsx title="src/components/StatusMessage.tsx"
import { Icon } from '@grasdouble/lufa_design-system';
import type { IconName } from '@grasdouble/lufa_design-system';

type StatusType = 'success' | 'error' | 'warning' | 'info';

interface StatusMessageProps {
  type: StatusType;
  message: string;
}

const statusConfig: Record<StatusType, { icon: IconName; color: ColorValue }> = {
  success: { icon: 'check-circle', color: 'success' },
  error: { icon: 'x-circle', color: 'error' },
  warning: { icon: 'alert-circle', color: 'warning' },
  info: { icon: 'info', color: 'info' },
};

export function StatusMessage({ type, message }: StatusMessageProps) {
  const { icon, color } = statusConfig[type];
  
  return (
    <div
      role="alert"
      aria-live="polite"
      style={{
        display: 'flex',
        alignItems: 'center',
        gap: '12px',
        padding: '12px 16px',
        borderRadius: '8px',
        border: `1px solid var(--lufa-token-color-border-${type})`,
        background: `var(--lufa-token-color-background-${type})`,
      }}
    >
      <Icon name={icon} size="md" color={color} />
      <span>{message}</span>
    </div>
  );
}

// Usage
<StatusMessage type="success" message="Operation completed successfully!" />
<StatusMessage type="error" message="An error occurred." />
```

### Pattern 3: Icon-Only Button

Create accessible icon-only buttons for toolbars and action menus.

```tsx title="src/components/IconButton.tsx"
import { Icon } from '@grasdouble/lufa_design-system';
import type { IconName } from '@grasdouble/lufa_design-system';

interface IconButtonProps {
  icon: IconName;
  label: string; // For accessibility
  onClick: () => void;
  variant?: 'default' | 'danger';
}

export function IconButton({ icon, label, onClick, variant = 'default' }: IconButtonProps) {
  return (
    <button
      onClick={onClick}
      aria-label={label}
      title={label}
      style={{
        padding: '8px',
        border: `1px solid ${variant === 'danger' ? 'var(--lufa-token-color-border-error)' : 'var(--lufa-token-color-border-default)'}`,
        borderRadius: '6px',
        background: 'var(--lufa-token-color-background-surface)',
        cursor: 'pointer',
      }}
    >
      <Icon 
        name={icon} 
        size="md" 
        color={variant === 'danger' ? 'error' : 'primary'} 
        title={label}
      />
    </button>
  );
}

// Usage
<div style={{ display: 'flex', gap: '8px' }}>
  <IconButton icon="edit" label="Edit item" onClick={handleEdit} />
  <IconButton icon="trash" label="Delete item" onClick={handleDelete} variant="danger" />
  <IconButton icon="settings" label="Settings" onClick={handleSettings} />
</div>
```

/**

export interface IconProps<T extends ElementType = 'span'> {

**Benefits:**

- **Type safety**: Icon names validated at compile time

- **Documentation**: Hover over props for JSDoc descriptions

**Example:**

```tsx
// ✅ Type-safe icon name

// ❌ Type error: invalid icon name

// ❌ Type error: href not valid for span/div
<Icon as="div" href="/page" name="star" />
```

## Troubleshooting

### Issue 1: Icon not displaying

**Problem**: Icon component renders but no icon appears.

**Solution**: Ensure the icon name is valid and part of the icon map.

```tsx
{/* ❌ Invalid icon name */}
<Icon name="invalid-name" />

{/* ✅ Valid icon name */}
<Icon name="user" />
```

### Issue 2: Icon color not changing

**Problem**: Setting `color` prop has no effect on icon color.

**Solution**: Ensure you're using semantic color values, not custom CSS colors.

```tsx
{/* ❌ Custom color doesn't work with color prop */}
<Icon name="heart" color="#FF0000" />

{/* ✅ Use semantic color */}
<Icon name="heart" color="error" />

{/* ✅ Or use style for custom colors */}
<Icon name="heart" style={{ color: '#FF0000' }} />
```

### Issue 3: Icon size not correct

**Problem**: Icon appears too small or too large.

**Solution**: Use appropriate size variant for context.

```tsx
{/* For inline with small text */}
<Icon name="info" size="xs" />

{/* For inline with body text */}
<Icon name="check" size="sm" />

{/* For standalone or buttons */}
<Icon name="star" size="md" />

{/* For hero sections */}
<Icon name="heart" size="xl" />
```

### Issue 4: Accessibility warnings

**Problem**: Screen reader warnings or failed accessibility audits.

**Solution**: Provide `title` for icon-only interactive elements, omit for decorative.

```tsx
{/* ❌ Icon-only button without title */}
<button>
  <Icon name="trash" />
</button>

{/* ✅ Icon-only button with title */}
<button>
  <Icon name="trash" title="Delete item" />
</button>

{/* ✅ Decorative icon with text label */}
<button>
  <Icon name="trash" size="sm" />
  <span>Delete</span>
</button>
```

### Issue 5: Icons not aligned with text

**Problem**: Icons don't align properly with adjacent text.

**Solution**: Icons use `vertical-align: middle` by default. Ensure proper flex container or adjust alignment.

```tsx
{/* ✅ Flex container for alignment */}
<div style={{ display: 'flex', alignItems: 'center', gap: '8px' }}>
  <Icon name="check" size="sm" />
  <span>Success message</span>
</div>

{/* Or use inline-flex on parent */}
<span style={{ display: 'inline-flex', alignItems: 'center', gap: '4px' }}>
  <Icon name="info" size="xs" />
  Information
</span>
```
