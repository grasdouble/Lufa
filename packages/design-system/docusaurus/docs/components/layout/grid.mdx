# Grid

import { DarkModeCompatible } from '../../../src/components/DarkModeCompatible';

<DarkModeCompatible />

A token-driven CSS grid layout with semantic column and gutter presets. Create responsive two-dimensional layouts with consistent spacing and flexible column configurations.

## Overview

Use `Grid` when you need…

- A simple, consistent CSS grid without custom templates
- Column presets via `columns`
- Gap presets via `gutter`
- Layouts like card lists, dashboards, and galleries

## Anatomy

The Grid component creates a two-dimensional CSS grid layout:

```
┌─────────────────────────────────────────────────────┐
│  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐             │
│  │ Col 1│  │ Col 2│  │ Col 3│  │ Col 4│             │  ← Row 1
│  └──────┘  └──────┘  └──────┘  └──────┘             │
│     ↕         ↕         ↕         ↕                  │
│  Gutter   Gutter   Gutter   Gutter                  │
│     ↕         ↕         ↕         ↕                  │
│  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐             │
│  │ Col 1│  │ Col 2│  │ Col 3│  │ Col 4│             │  ← Row 2
│  └──────┘  └──────┘  └──────┘  └──────┘             │
└─────────────────────────────────────────────────────┘
     ←──────────────────────────────→
              Grid Container
```

**Component Structure:**
- **Grid Container**: The outer grid wrapper with `display: grid`
- **Grid Items**: Direct children automatically placed in grid cells
- **Columns**: Number of vertical divisions (1, 2, 3, 4, 6, 8, 12, or 16)
- **Gutter**: Spacing between grid items (horizontal and vertical gap)
- **Auto-flow**: Items flow left-to-right, top-to-bottom by default

**Layout Behavior:**
- Items automatically wrap to new rows when columns are filled
- Equal-width columns using `repeat(N, 1fr)`
- Consistent gutter spacing from design tokens
- Responsive by default (items stack on smaller screens)

## API Reference

### Props

| Prop       | Type                        | Default    | Description                      |
| ---------- | --------------------------- | ---------- | -------------------------------- |
| `columns`  | `keyof typeof GRID_COLUMNS` | `'twelve'` | Number of columns (semantic token) |
| `gutter`   | `keyof typeof GRID_GUTTER`  | `'md'`     | Gap between grid items (token)   |
| `children` | `ReactNode`                 | -          | Grid items (direct children)     |
| `className`| `string`                    | -          | Additional CSS class             |

Also supports all standard HTML `div` element attributes.

### Columns Details

- **single**: 1 column (full width items)
- **double**: 2 columns (50% width each)
- **triple**: 3 columns (33.33% width each)
- **quad**: 4 columns (25% width each)
- **six**: 6 columns (16.67% width each)
- **eight**: 8 columns (12.5% width each)
- **twelve**: 12 columns (8.33% width each) - default, flexible grid system
- **sixteen**: 16 columns (6.25% width each)

### Gutter Details

- **none**: No spacing between items (0px)
- **xs**: Extra small spacing (4px)
- **sm**: Small spacing (8px)
- **md**: Medium spacing (16px) - default, balanced spacing
- **lg**: Large spacing (24px)
- **xl**: Extra large spacing (32px)

## Best Practices

### Do ✅

- Use token presets (`GRID_COLUMNS`, `GRID_GUTTER`) for consistent layout rhythm
- Combine `Grid` with `Container` to keep wide layouts readable
- Use `gutter="none"` for dense layouts only when spacing is handled elsewhere

### Don't ❌

- Don’t use `Grid` for 1D layouts; prefer `Flex`/`Stack`
- Don’t mix inconsistent gutters across similar grids in the same view

## Accessibility

The Grid component is a layout primitive designed with WCAG 2.1 AA compliance in mind:

### Keyboard Navigation

Grid is a layout-only component and does not implement keyboard navigation directly. Keyboard interaction depends on the interactive elements within grid items.

**Best Practices:**
- Ensure interactive grid items (cards, buttons, links) have visible focus indicators
- Maintain logical tab order (left-to-right, top-to-bottom)
- Use consistent focus order across responsive breakpoints
- Consider using `tabindex="0"` for focusable card containers

### Screen Reader Support

Grid uses a standard div element and relies on semantic HTML within grid items:

**Basic Grid (Non-semantic):**
```tsx
<Grid columns={GRID_COLUMNS.triple} gutter={GRID_GUTTER.md}>
  <div>Item 1</div>
  <div>Item 2</div>
  <div>Item 3</div>
</Grid>
// No special announcement, reads items in order
```

**Grid with Semantic Items:**
```tsx
<Grid columns={GRID_COLUMNS.quad} gutter={GRID_GUTTER.lg}>
  <article aria-labelledby="card-1">
    <h3 id="card-1">Product 1</h3>
    <p>Description</p>
  </article>
  <article aria-labelledby="card-2">
    <h3 id="card-2">Product 2</h3>
    <p>Description</p>
  </article>
</Grid>
// Each article announces: "Article, Product 1" followed by content
```

**Grid with ARIA Label:**
```tsx
<Grid
  columns={GRID_COLUMNS.triple}
  gutter={GRID_GUTTER.md}
  role="region"
  aria-label="Product gallery"
>
  {/* grid items */}
</Grid>
// Announces: "Product gallery, region" followed by item content
```

### Best Practices for Accessibility

#### Do ✅

- Use semantic HTML within grid items (article, section, heading structure)
- Provide descriptive labels for interactive grid items
- Ensure sufficient color contrast for grid item content
- Maintain consistent tab order across responsive layouts
- Use meaningful alt text for images in grid items
- Ensure minimum 44×44px touch targets for mobile interactive items
- Test with keyboard-only navigation

#### Don't ❌

- Don't rely solely on visual grid layout to convey relationships
- Don't create overly complex nesting that confuses screen readers
- Don't use Grid for data tables (use Table component instead)
- Don't remove focus indicators from grid items
- Don't reorder grid items visually without considering reading order
- Don't use small gutters that make touch targets too close

### ARIA Attributes

The Grid component itself doesn't apply ARIA attributes by default. When needed, pass them as props:

```tsx
<Grid
  columns={GRID_COLUMNS.quad}
  gutter={GRID_GUTTER.md}
  role="list"
  aria-label="Featured products"
>
  <div role="listitem">{/* product */}</div>
  <div role="listitem">{/* product */}</div>
  <div role="listitem">{/* product */}</div>
</Grid>
```

For grouped content:
```tsx
<Grid
  columns={GRID_COLUMNS.triple}
  gutter={GRID_GUTTER.lg}
  role="region"
  aria-labelledby="section-title"
>
  <h2 id="section-title">Product Categories</h2>
  {/* category items */}
</Grid>
```

### Color Contrast

Grid is a transparent layout component. Color contrast requirements apply to its children:
- Text content: Minimum 4.5:1 contrast ratio (WCAG AA)
- Interactive elements: Minimum 3:1 contrast for UI components
- Focus indicators: Minimum 3:1 contrast against background
- Card borders/backgrounds: Ensure sufficient contrast with page background

## Common Patterns

### Image Gallery

```tsx title="src/components/ImageGallery.tsx"
import { Grid, GRID_COLUMNS, GRID_GUTTER } from '@grasdouble/lufa_design-system';

function ImageGallery({ images }: { images: ImageData[] }) {
  return (
    <Grid
      columns={GRID_COLUMNS.quad}
      gutter={GRID_GUTTER.md}
      role="region"
      aria-label="Photo gallery"
    >
      {images.map((image) => (
        <div key={image.id} className="aspect-square">
          <img
            src={image.url}
            alt={image.description}
            className="w-full h-full object-cover rounded"
          />
        </div>
      ))}
    </Grid>
  );
}
```

### Responsive Card Grid

```tsx title="src/components/ProductGrid.tsx"
import { Grid, GRID_COLUMNS, GRID_GUTTER } from '@grasdouble/lufa_design-system';

function ProductGrid({ products }: { products: Product[] }) {
  return (
    <Grid
      columns={GRID_COLUMNS.quad}
      gutter={GRID_GUTTER.lg}
      className="responsive-grid" // Custom responsive breakpoints
    >
      {products.map((product) => (
        <article key={product.id} className="product-card">
          <img src={product.image} alt={product.name} />
          <h3>{product.name}</h3>
          <p>{product.price}</p>
          <Button>Add to Cart</Button>
        </article>
      ))}
    </Grid>
  );
}

// CSS for responsive breakpoints
// .responsive-grid {
//   @media (max-width: 768px) { grid-template-columns: repeat(2, 1fr); }
//   @media (max-width: 480px) { grid-template-columns: 1fr; }
// }
```

### Product Listing

```tsx title="src/components/ProductListing.tsx"
import { Grid, GRID_COLUMNS, GRID_GUTTER } from '@grasdouble/lufa_design-system';

function ProductListing({ products }: { products: Product[] }) {
  return (
    <div>
      <h2>Featured Products</h2>
      <Grid
        columns={GRID_COLUMNS.triple}
        gutter={GRID_GUTTER.xl}
        role="list"
        aria-label="Featured products"
      >
        {products.map((product) => (
          <article key={product.id} role="listitem" className="product-item">
            <div className="product-image">
              <img src={product.thumbnail} alt={product.name} />
              {product.badge && <Badge>{product.badge}</Badge>}
            </div>
            <h3>{product.name}</h3>
            <p className="description">{product.description}</p>
            <div className="price-and-action">
              <span className="price">{product.price}</span>
              <Button variant="primary">View Details</Button>
            </div>
          </article>
        ))}
      </Grid>
    </div>
  );
}
```

## TypeScript Support

The Grid component is fully typed with TypeScript:

```tsx
import type { ReactNode, HTMLAttributes } from 'react';

export type GridProps = {
  /** Grid content */
  children?: ReactNode;
  /** Number of columns (semantic tokens) */
  columns?: keyof typeof GRID_COLUMNS;
  /** Spacing between grid items */
  gutter?: keyof typeof GRID_GUTTER;
} & HTMLAttributes<HTMLDivElement>;

// Constants
export const GRID_COLUMNS = {
  single: 'single',
  double: 'double',
  triple: 'triple',
  quad: 'quad',
  six: 'six',
  eight: 'eight',
  twelve: 'twelve',
  sixteen: 'sixteen',
} as const;

export const GRID_GUTTER = {
  none: 'none',
  xs: 'xs',
  sm: 'sm',
  md: 'md',
  lg: 'lg',
  xl: 'xl',
} as const;
```

This ensures type safety and autocomplete in your IDE.
