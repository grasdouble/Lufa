# Space

<DarkModeCompatible />

A simple spacer element for adding fixed vertical or horizontal space. Uses design tokens by default for consistent spacing rhythm.

## Overview

Use `Space` when you need…

- Explicit spacing without changing layout structure
- Vertical or horizontal spacing via `direction`
- Token presets or custom values via `size`
- Render element control via `as` (`div`/`span`)

## Basic Usage

```tsx title="src/App.tsx"
import { Space, SPACE_DIRECTION, SPACE_SIZE } from '@grasdouble/lufa_design-system';

function App() {
  return (
    <>
      {/* Vertical spacing */}
      <div>Before</div>
      <Space size={SPACE_SIZE.lg} />
      <div>After</div>

      {/* Horizontal spacing */}
      <div style={{ display: "flex", alignItems: "center" }}>
        <div>Left</div>
        <Space direction={SPACE_DIRECTION.horizontal} size={SPACE_SIZE.md} />
        <div>Right</div>
      </div>
    </>
  );
}
```

## Anatomy

The Space component creates invisible spacing:

```
Vertical:                    Horizontal:
┌─────────┐                 ┌───┐ ⇿ ┌───┐
│ Content │                 │ A │   │ B │
└─────────┘                 └───┘   └───┘
     ⇳                         ↑       ↑
  [Space]                   Content Space Content
     ⇳
┌─────────┐
│ Content │
└─────────┘
```

**Component Structure:**
- **Container**: An empty `div` or `span` element (controlled by `as` prop)
- **Direction**: Vertical (default) or horizontal spacing
- **Size**: Token-based (xs, sm, md, lg, xl) or custom value (number/string)

**Rendering:**
- Default renders as `<div>` with block display
- Use `as="span"` for inline spacing within text

## API Reference

### Props

| Prop        | Type                                          | Default      | Description      |
| ----------- | --------------------------------------------- | ------------ | ---------------- |
| `as`        | `'div' \| 'span'`                             | `'div'`      | Render element   |
| `direction` | `keyof typeof SPACE_DIRECTION`                | `'vertical'` | Spacer direction |
| `size`      | `keyof typeof SPACE_SIZE \| number \| string` | `'md'`       | Spacer size      |

Also supports standard element props (for example `className`, `style`, `aria-label`, …).

### Direction Details

- **vertical** (default): Creates vertical spacing using margin-top
- **horizontal**: Creates horizontal spacing using margin-left, useful in flex layouts

### Size Details

Token-based sizes map to the design system spacing scale:

- **xs**: Extra small spacing (4px)
- **sm**: Small spacing (8px)
- **md**: Medium spacing (16px, default)
- **lg**: Large spacing (24px)
- **xl**: Extra large spacing (32px)

Custom sizes can be provided as:
- **number**: Interpreted as pixels (e.g., `size={20}` = 20px)
- **string**: Any CSS length value (e.g., `size="2rem"`)

### As Details

- **div** (default): Block-level element, creates new line
- **span**: Inline element, useful for spacing within text or flex containers

## Best Practices

### Do ✅

- Use `Space` for small, explicit gaps where changing markup isn’t worth it
- Prefer token sizes (`SPACE_SIZE`) for consistent rhythm
- Use `as="span"` for inline spacing in text

### Don't ❌

- Don’t overuse spacers for layout; prefer `Stack`, `Flex`, or `Grid`
- Don’t rely on `Space` for semantic separation; it’s purely visual/layout

## Accessibility

The Space component is a layout utility with built-in accessibility considerations:

### Built-in Features

- **aria-hidden="true"**: Applied by default since Space is purely decorative
- **Non-interactive**: Space should never contain interactive content

### Best Practices for Accessibility

#### Do ✅

- Use Space for visual spacing only
- Apply padding/margins directly to interactive elements for hit targets
- Use semantic spacing components (Stack, Flex) for structural layouts
- Prefer token sizes for consistent visual rhythm

#### Don't ❌

- Don't use Space to create click targets (impacts touch/click areas)
- Don't place interactive elements inside Space
- Don't use excessive spacing values that break visual hierarchy
- Don't rely on Space alone for layout structure

### Screen Reader Support

Space is automatically hidden from screen readers (`aria-hidden="true"`), ensuring assistive technologies focus on actual content rather than layout utilities.

```tsx
// Space is decorative and automatically hidden from screen readers
<div>Content A</div>
<Space size="lg" />  {/* Hidden from screen readers */}
<div>Content B</div>
```

## Common Patterns

### Vertical Content Spacing

```tsx title="src/components/Article.tsx"
import { Space, SPACE_SIZE } from '@grasdouble/lufa_design-system';

function Article() {
  return (
    <article>
      <h1>Article Title</h1>
      <Space size={SPACE_SIZE.md} />
      <p>First paragraph...</p>
      <Space size={SPACE_SIZE.sm} />
      <p>Second paragraph...</p>
      <Space size={SPACE_SIZE.lg} />
      <footer>Published on...</footer>
    </article>
  );
}
```

### Inline Spacing in Text

```tsx title="src/components/Badge.tsx"
import { Space, SPACE_SIZE, SPACE_DIRECTION } from '@grasdouble/lufa_design-system';

function BadgeList() {
  return (
    <div>
      <span className="badge">React</span>
      <Space as="span" direction={SPACE_DIRECTION.horizontal} size={SPACE_SIZE.sm} />
      <span className="badge">TypeScript</span>
      <Space as="span" direction={SPACE_DIRECTION.horizontal} size={SPACE_SIZE.sm} />
      <span className="badge">Design System</span>
    </div>
  );
}
```

### Custom Spacing Values

```tsx title="src/components/CustomLayout.tsx"
import { Space, SPACE_DIRECTION } from '@grasdouble/lufa_design-system';

function CustomLayout() {
  return (
    <div style={{ display: 'flex' }}>
      <div>Sidebar</div>
      {/* Custom pixel value */}
      <Space direction={SPACE_DIRECTION.horizontal} size={48} />
      <div>Main Content</div>
    </div>
  );
}
```

## TypeScript Support

The Space component is fully typed with TypeScript:

```tsx
import type { HTMLAttributes } from 'react';

type SpaceSize = keyof typeof SPACE_SIZE | number;
type SpaceDirection = keyof typeof SPACE_DIRECTION;
type SpaceElement = 'div' | 'span';

interface SpaceProps extends HTMLAttributes<HTMLElement> {
  as?: SpaceElement;
  direction?: SpaceDirection;
  size?: SpaceSize;
}

// Constants for type safety
const SPACE_SIZE = {
  xs: 'xs',
  sm: 'sm',
  md: 'md',
  lg: 'lg',
  xl: 'xl',
} as const;

const SPACE_DIRECTION = {
  vertical: 'vertical',
  horizontal: 'horizontal',
} as const;
```

This ensures type safety and autocomplete in your IDE.
