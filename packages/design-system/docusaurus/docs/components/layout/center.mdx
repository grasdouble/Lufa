# Center

import { DarkModeCompatible } from '../../../src/components/DarkModeCompatible';

<DarkModeCompatible />

Centers children horizontally, vertically, or both using flex (optionally inline).

## Overview

Use `Center` when you need…

- Quick centering without custom flex CSS
- Axis control via `axis` (`horizontal`, `vertical`, `both`)
- Vertical centering via `minHeight`
- Inline flow support via `inline` (`inline-flex`)

## Import


## Basic Usage

```tsx
<Center axis="both" minHeight={240}>
  <div>Centered</div>
</Center>
```

## Anatomy

The Center component is a flex layout utility:

```
┌─────────────────────┐
│                     │
│    [Centered]       │  ← Child centered based on axis
│                     │
└─────────────────────┘

axis="both":        axis="horizontal":   axis="vertical":
     ○                    ○               ○
  centered            centered left       top centered
                          to right
```

**Component Structure:**
- **Flex Container**: Uses `display: flex` (or `inline-flex`) with centering
- **Alignment**: `justify-content` and/or `align-items` based on `axis`
- **Semantic Element**: Renders as configurable element via `as` prop

**Visual States:**
- **Both**: Centers horizontally and vertically (default)
- **Horizontal**: Centers horizontally only (children stack vertically)
- **Vertical**: Centers vertically only (children flow horizontally)
- **Inline**: Uses `inline-flex` instead of `flex`

## API Reference

### Props

| Prop        | Type                                                  | Default  | Description                               |
| ----------- | ----------------------------------------------------- | -------- | ----------------------------------------- |
| `axis`      | `'both' \| 'horizontal' \| 'vertical'`                | `'both'` | Which axis to center on                   |
| `inline`    | `boolean`                                             | `false`  | Uses `inline-flex` instead of `flex`      |
| `minHeight` | `number \| string`                                    | -        | Optional minimum height for the container |
| `as`        | `'div' \| 'span' \| 'section' \| 'main' \| 'article'` | `'div'`  | Render element                            |
| `children`  | `ReactNode`                                           | -        | Content                                   |

Also supports standard element props (for example `className`, `style`, `aria-label`, …).

## Best Practices

### Do ✅

- Use `minHeight` when demonstrating or requiring vertical centering
- Use `inline` + `as="span"` when centering content inside text flow
- Prefer semantic `as` values (`main`, `section`, `article`) when the container has meaning

### Don't ❌

- Don’t use `Center` for complex layouts; use `Stack`/grid instead
- Don’t rely on `Center` to “fix” spacing issues; ensure container sizing is intentional

## Accessibility

The Center component is designed with WCAG 2.1 AA compliance in mind:

### Screen Reader Support

Center is a layout-only component and does not add semantic meaning. Screen readers will announce the content based on the `as` element and children.

```tsx
<Center as="section" aria-label="Loading state">
  <Spinner />
  <p>Loading content...</p>
</Center>
// Screen reader announces: "Loading state, region" then reads content
```

### Best Practices for Accessibility

#### Do ✅

- Use semantic `as` values when the container has meaning (e.g., `section`, `main`)
- Ensure centered content has sufficient contrast
- Use `minHeight` judiciously to avoid excessive white space
- Provide context for centered content (headings, labels)

#### Don't ❌

- Don't center critical content without surrounding context
- Don't use Center to hide or obscure content
- Don't rely solely on visual centering to convey importance
- Don't use excessive `minHeight` that creates scrolling issues

### ARIA Attributes

The component supports standard HTML attributes based on the `as` element:

```tsx
<Center as="section" aria-label="Empty state">
  <p>No items found</p>
</Center>
```

### Color Contrast

Center is a transparent layout container. Ensure child content meets WCAG 2.1 AA contrast requirements (4.5:1 for normal text, 3:1 for large text).

## Common Patterns

### Loading State

```tsx title="src/components/LoadingState.tsx"
import { Center, Spinner } from '@grasdouble/lufa_design-system';

function LoadingState({ message = 'Loading...' }) {
  return (
    <Center axis="both" minHeight={200}>
      <div style={{ textAlign: 'center' }}>
        <Spinner size="large" />
        <p style={{ marginTop: '1rem' }}>{message}</p>
      </div>
    </Center>
  );
}
```

### Empty State

```tsx title="src/components/EmptyState.tsx"
import { Center } from '@grasdouble/lufa_design-system';

function EmptyState({ title, description, action }) {
  return (
    <Center axis="both" minHeight={300}>
      <div style={{ textAlign: 'center', maxWidth: '400px' }}>
        <h3>{title}</h3>
        <p style={{ color: 'var(--color-text-secondary)' }}>
          {description}
        </p>
        {action}
      </div>
    </Center>
  );
}
```

### Inline Centered Badge

```tsx title="src/components/StatusBadge.tsx"
import { Center } from '@grasdouble/lufa_design-system';

function StatusBadge({ label }) {
  return (
    <Center
      as="span"
      inline
      axis="both"
      style={{
        width: '24px',
        height: '24px',
        borderRadius: '50%',
        background: 'var(--color-primary)',
        color: 'white',
        fontSize: '12px',
      }}
    >
      {label}
    </Center>
  );
}
```

## TypeScript Support

The Center component is fully typed with TypeScript:

```tsx
import type { ElementType, HTMLAttributes, ReactNode } from 'react';

export type CenterAxis = 'both' | 'horizontal' | 'vertical';
type CenterElement = 'div' | 'span' | 'section' | 'main' | 'article';

export type CenterProps = {
  as?: CenterElement;
  axis?: CenterAxis;
  inline?: boolean;
  minHeight?: number | string;
  children?: ReactNode;
} & HTMLAttributes<HTMLElement>;
```

This ensures type safety and autocomplete in your IDE.
