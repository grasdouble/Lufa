# Tabs

Switchable views for grouping related content. Organize information into multiple panels that users can navigate between without leaving the current page.

## Overview

Use `Tabs` when you need to organize related content into switchable panels.

- Define tabs via `items` (label + optional icon/disabled + panel content)
- Choose a visual style with `type`
- Control placement with `tabPosition`
- Use controlled (`activeKey` + `onChange`) or uncontrolled (`defaultActiveKey`) mode

## Basic Usage

```tsx
<Tabs
  items={[
    { key: 'tab1', label: 'Tab 1', children: <div>Content 1</div> },
    { key: 'tab2', label: 'Tab 2', children: <div>Content 2</div> },
    { key: 'tab3', label: 'Tab 3', children: <div>Content 3</div> },
  ]}
/>
```

## Anatomy

The Tabs component consists of several key parts:

```
┌─────────────────────────────────────────┐
│  [Tab 1]  [Tab 2]  [Tab 3]              │  ← Tab List
│  ═══════                                │  ← Active Indicator
├─────────────────────────────────────────┤
│                                         │
│  Content for active tab panel           │  ← Tab Panel
│                                         │
└─────────────────────────────────────────┘
     ↑         ↑         ↑
     │         │         └─ Inactive tab
     │         └─────────── Inactive tab
     └─────────────────── Active tab
```

**Component Structure:**
- **Tab List**: Container for all tab buttons with horizontal or vertical orientation
- **Tab Buttons**: Clickable labels for switching between panels
- **Active Indicator**: Visual highlight (underline, background, or pill) showing the selected tab
- **Tab Panel**: Content area displaying the active tab's children
- **Optional Icons**: Leading icons before tab labels

**Visual States:**
- **Default**: Normal interactive state for inactive tabs
- **Hover**: Visual feedback on mouse over
- **Focus**: Keyboard navigation indicator (focus ring)
- **Active**: Selected tab with visual emphasis (underline/background/pill)
- **Disabled**: Reduced opacity, not focusable, no interaction

## API Reference

### Props

| Prop               | Type                                     | Default    | Description                           |
| ------------------ | ---------------------------------------- | ---------- | ------------------------------------- |
| `items`            | `TabItem[]`                              | -          | Array of tab items                    |
| `activeKey`        | `string`                                 | -          | Active tab key (controlled)           |
| `defaultActiveKey` | `string`                                 | -          | Default active tab key (uncontrolled) |
| `onChange`         | `(key: string) => void`                  | -          | Tab change handler                    |
| `type`             | `'line' \| 'card' \| 'pill'`             | `'line'`   | Tab visual type                       |
| `tabPosition`      | `'top' \| 'bottom' \| 'left' \| 'right'` | `'top'`    | Tab position                          |
| `size`             | `'small' \| 'medium' \| 'large'`         | `'medium'` | Size variant                          |
| `className`        | `string`                                 | -          | Additional CSS class                  |

### TabItem

| Property   | Type        | Description                  |
| ---------- | ----------- | ---------------------------- |
| `key`      | `string`    | Unique identifier            |
| `label`    | `string`    | Tab label                    |
| `children` | `ReactNode` | Tab content (optional)       |
| `icon`     | `ReactNode` | Icon before label (optional) |
| `disabled` | `boolean`   | Disabled state (optional)    |

Also supports all standard HTML `div` attributes for the container element.

### Type Details

- **line**: Minimal underline indicator beneath active tab (default, clean and subtle)
- **card**: Tab buttons with card/segment styling (good for settings pages)
- **pill**: Rounded pill-shaped background for active tab (modern, playful)

### Tab Position Details

- **top**: Tabs above content (default, most common)
- **bottom**: Tabs below content (rare, specific use cases)
- **left**: Vertical tabs on left side (good for sidebars)
- **right**: Vertical tabs on right side (alternative sidebar placement)

### Size Details

- **small**: Compact size for dense interfaces (28px height)
- **medium**: Default size for most use cases (36px height)
- **large**: Prominent size for primary navigation (44px height)

## Best Practices

### Do ✅

- Use short labels (1–2 words) and keep the number of tabs reasonable
- Prefer `type="card"` for settings pages and `type="line"` for content sections
- Use disabled tabs to communicate unavailable sections (and explain why in the UI)

### Don't ❌

- Don’t use tabs to hide critical information required to complete a task
- Don’t mix navigation (route changes) and in-page tabs without clear UX cues
- Don’t overload tab panels with long-form content (consider separate pages)

## Accessibility

The Tabs component is designed with WCAG 2.1 AA compliance in mind:

### Keyboard Navigation

- **Arrow Right/Down**: Navigate to next tab (wraps to first)
- **Arrow Left/Up**: Navigate to previous tab (wraps to last)
- **Home**: Jump to first enabled tab
- **End**: Jump to last enabled tab
- **Tab**: Move focus to tab panel content
- **Shift + Tab**: Return to active tab button

Disabled tabs are automatically skipped during keyboard navigation.

### Screen Reader Support

**For Standard Tabs:**
```tsx
<Tabs
  items={[
    { key: 'profile', label: 'Profile', children: <ProfileContent /> },
    { key: 'settings', label: 'Settings', children: <SettingsContent /> },
  ]}
/>
// Active tab announces: "Profile, tab, 1 of 2, selected"
// Inactive tab announces: "Settings, tab, 2 of 2"
```

**For Tabs with Icons:**
```tsx
<Tabs
  items={[
    { key: 'home', label: 'Home', icon: <HomeIcon />, children: <HomeContent /> },
    { key: 'user', label: 'Profile', icon: <UserIcon />, children: <ProfileContent /> },
  ]}
/>
// Screen reader announces: "Home, tab, 1 of 2, selected" (icon is decorative)
```

**For Disabled Tabs:**
```tsx
<Tabs
  items={[
    { key: 'tab1', label: 'Available', children: <Content1 /> },
    { key: 'tab2', label: 'Disabled', disabled: true },
  ]}
/>
// Disabled tab announces: "Disabled, tab, 2 of 2, unavailable"
```

### Best Practices for Accessibility

#### Do ✅

- Keep tab labels short and descriptive (1-2 words)
- Provide meaningful panel content for each tab
- Use `disabled` with explanation in UI for unavailable tabs
- Ensure tab order is logical left-to-right (or top-to-bottom for vertical)
- Keep focus indicators visible (default focus ring)
- Use semantic heading structure within tab panels

#### Don't ❌

- Don't hide critical information in tabs (should be immediately visible)
- Don't use too many tabs (4-7 is ideal, more requires scrolling)
- Don't change tab order dynamically (confusing for keyboard users)
- Don't remove focus outlines (breaks keyboard navigation)
- Don't use tabs for step-by-step processes (use Steps component instead)

### ARIA Attributes

The component automatically handles:
- `role="tablist"` on the tab container
- `role="tab"` on each tab button
- `role="tabpanel"` on each content panel
- `aria-selected="true"` on the active tab
- `aria-controls` linking tab to its panel
- `aria-labelledby` linking panel to its tab
- `aria-disabled="true"` on disabled tabs
- `aria-orientation="horizontal"` or `"vertical"` based on `tabPosition`
- `tabindex="0"` on active tab, `tabindex="-1"` on inactive tabs

Additional ARIA attributes can be passed as props:

```tsx
<Tabs
  items={items}
  aria-label="User settings navigation"
/>
```

### Color Contrast

All tab variants meet WCAG 2.1 AA contrast requirements:
- Tab labels: Minimum 4.5:1 contrast ratio
- Active indicator: Minimum 3:1 contrast ratio
- Disabled tabs: Reduced opacity while maintaining readable contrast

Tested in both light and dark modes.

### Controlled Tabs with State

```tsx title="src/components/ControlledTabs.tsx"
import { Tabs } from '@grasdouble/lufa_design-system';
import { useState } from 'react';

function ControlledTabs() {
  const [activeTab, setActiveTab] = useState('overview');

  const handleTabChange = (key: string) => {
    setActiveTab(key);
    // Log analytics, update URL, etc.
    console.log('Tab changed to:', key);
  };

  return (
    <Tabs
      activeKey={activeTab}
      onChange={handleTabChange}
      items={[
        { key: 'overview', label: 'Overview', children: <Overview /> },
        { key: 'details', label: 'Details', children: <Details /> },
        { key: 'reviews', label: 'Reviews', children: <Reviews /> },
      ]}
    />
  );
}
```

function ConditionalTabs({ isPremiumUser }: { isPremiumUser: boolean }) {

## TypeScript Support

The Tabs component is fully typed with TypeScript:

```tsx
import type { ReactNode } from 'react';

export type TabItem = {
  key: string;
  label: string;
  children?: ReactNode;
  icon?: ReactNode;
  disabled?: boolean;
};

export type TabsProps = {
  items: TabItem[];
  activeKey?: string;
  defaultActiveKey?: string;
  onChange?: (key: string) => void;
  type?: 'line' | 'card' | 'pill';
  tabPosition?: 'top' | 'bottom' | 'left' | 'right';
  size?: 'small' | 'medium' | 'large';
  className?: string;
};
```

This ensures type safety and autocomplete in your IDE.
