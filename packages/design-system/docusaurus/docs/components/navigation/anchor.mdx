# Anchor

<DarkModeCompatible />

In-page navigation link for jumping to a section on the same page. A specialized link component optimized for anchor navigation within a single document.

## Overview

Use `Anchor` when you need…

- In-page navigation (jump to a section)
- Visual variants via `variant`
- Color variants via `color`
- Optional `startIcon` / `endIcon`

## Anatomy

The Anchor component creates in-page navigation links:

```
┌─────────────────────────────────────┐
│  [startIcon]  Label  [endIcon]      │  ← Anchor Container
└─────────────────────────────────────┘
     ↑          ↑         ↑
     │          │         └─ Optional trailing icon
     │          └────────────── Required text content
     └───────────────────────── Optional leading icon
```

**Component Structure:**
- **Container**: Native `<a>` element with in-page navigation styling
- **Start Icon** (optional): Leading icon positioned before the label
- **Label**: The main anchor text (required for accessibility)
- **End Icon** (optional): Trailing icon positioned after the label

**Visual States:**
- Default: Styled based on variant (default, underline, subtle)
- Hover: Color change and visual feedback
- Focus: Keyboard navigation indicator (focus ring)
- Active: Currently viewing section (requires custom implementation)

**Typical Use Cases:**
- Table of contents navigation
- "Back to top" links
- Sidebar navigation within long documents
- Skip links for accessibility

## API Reference

### Props

| Prop        | Type                                    | Default     | Description                             |
| ----------- | --------------------------------------- | ----------- | --------------------------------------- |
| `children`  | `ReactNode`                             | -           | Anchor content                          |
| `variant`   | `'default' \| 'underline' \| 'subtle'`  | `'default'` | Visual style variant                    |
| `color`     | `'primary' \| 'secondary' \| 'inherit'` | `'primary'` | Color scheme                            |
| `startIcon` | `ReactNode`                             | -           | Icon before content                     |
| `endIcon`   | `ReactNode`                             | -           | Icon after content                      |
| `href`      | `string`                                | -           | Target anchor link (e.g. `#section-id`) |

Also supports standard anchor props (for example `target`, `rel`, `className`, `aria-*`, …).

### Variant Details

- **default**: Standard anchor styling with subtle color
- **underline**: Classic underlined link for maximum affordance
- **subtle**: Minimal styling, inherits most text properties

### Color Details

- **primary**: Main brand color, use for primary in-page navigation
- **secondary**: Neutral alternative, use for less prominent anchors
- **inherit**: Inherits color from parent, useful for matching surrounding text

## Best Practices

### Do ✅

- Use `Anchor` for in-page navigation (`href="#id"`) when the destination exists on the page
- Use `variant="underline"` when you want stronger link affordance
- Use `target="_blank"` + `rel="noreferrer"` (or `noopener`) for external links
- Ensure target IDs exist in the document
- Use descriptive labels that indicate the destination section

### Don't ❌

- Don't use anchors as buttons for actions; use `Button` for actions
- Don't link to missing IDs; it creates a broken experience
- Don't rely solely on color to distinguish anchors from regular text

## Accessibility

The Anchor component follows WCAG 2.1 AA guidelines for in-page navigation:

### Keyboard Navigation

- **Tab**: Navigate to anchor
- **Enter**: Jump to target section
- **Shift+Tab**: Navigate backward

All anchors are keyboard accessible by default with visible focus indicators.

### Screen Reader Support

**For Basic Anchors:**
```tsx
<Anchor href="#features">Jump to Features</Anchor>
// Screen reader announces: "Jump to Features, link"
```

**For Table of Contents:**
```tsx
<nav aria-label="Table of contents">
  <Anchor href="#intro">Introduction</Anchor>
  <Anchor href="#methods">Methods</Anchor>
  <Anchor href="#results">Results</Anchor>
</nav>
// Screen reader announces navigation landmark with links
```

**For Skip Links:**
```tsx
<Anchor href="#main-content" className="skip-link">
  Skip to main content
</Anchor>
// Provides quick navigation for keyboard/screen reader users
```

### Best Practices for Accessibility

#### Do ✅

- Provide clear link text that describes the destination ("Jump to Features" not "Click here")
- Use skip links to help keyboard users navigate efficiently
- Ensure target sections have matching IDs
- Consider smooth scrolling with `scroll-behavior: smooth`
- Use semantic HTML for target sections (headings, sections)

#### Don't ❌

- Don't use vague anchor text ("See below", "More info")
- Don't remove focus outlines (breaks keyboard navigation)
- Don't create dead links to non-existent IDs
- Don't nest interactive elements inside anchors

### ARIA Attributes

For enhanced navigation context:

```tsx
<Anchor
  href="#section"
  aria-label="Navigate to section details"
  aria-describedby="section-summary"
>
  Section
</Anchor>
```

### Color Contrast

All anchor variants meet WCAG 2.1 AA contrast requirements:
- Normal text: Minimum 4.5:1 contrast ratio
- Large text: Minimum 3:1 contrast ratio
- Interactive elements: Minimum 3:1 contrast ratio

Tested in both light and dark modes.

## Common Patterns

### Table of Contents

```tsx title="src/components/TableOfContents.tsx"
import { Anchor } from '@grasdouble/lufa_design-system';

function TableOfContents() {
  return (
    <nav aria-label="Table of contents">
      <h2>Contents</h2>
      <ul>
        <li><Anchor href="#introduction">Introduction</Anchor></li>
        <li><Anchor href="#getting-started">Getting Started</Anchor></li>
        <li><Anchor href="#advanced">Advanced Topics</Anchor></li>
        <li><Anchor href="#conclusion">Conclusion</Anchor></li>
      </ul>
    </nav>
  );
}
```

### Back to Top Link

```tsx title="src/components/BackToTop.tsx"
import { Anchor } from '@grasdouble/lufa_design-system';
import { ArrowUpIcon } from './icons';

function BackToTop() {
  return (
    <Anchor
      href="#top"
      variant="subtle"
      startIcon={<ArrowUpIcon />}
    >
      Back to top
    </Anchor>
  );
}
```

### Skip Navigation Link

```tsx title="src/components/SkipLink.tsx"
import { Anchor } from '@grasdouble/lufa_design-system';

function SkipLink() {
  return (
    <Anchor
      href="#main-content"
      className="skip-link"
      variant="default"
    >
      Skip to main content
    </Anchor>
  );
}

// CSS: .skip-link { position: absolute; top: -40px; }
// .skip-link:focus { top: 0; }
```

## TypeScript Support

The Anchor component is fully typed with TypeScript:

```tsx
import type { AnchorHTMLAttributes, ReactNode } from 'react';

interface AnchorProps extends AnchorHTMLAttributes<HTMLAnchorElement> {
  children?: ReactNode;
  variant?: 'default' | 'underline' | 'subtle';
  color?: 'primary' | 'secondary' | 'inherit';
  startIcon?: ReactNode;
  endIcon?: ReactNode;
}
```

This ensures type safety and autocomplete in your IDE.
