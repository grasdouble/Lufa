# Steps

<DarkModeCompatible />

Progress indicator for multi-step flows. Visualizes sequential processes like checkout, onboarding, or multi-part forms.

## Overview

Use `Steps` when you need to display progress through a multi-step flow.

- Steps are defined by `items` (title, optional description/icon/status)
- The active step is controlled by `current` (0-indexed)
- Layout is controlled by `direction` (horizontal/vertical)
- Provide `onChange` to make steps clickable

## Anatomy

The Steps component visualizes progress through a sequence:

```
Horizontal Layout:
┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐
│ ✓       │─────│ 2       │─────│ 3       │─────│ 4       │
│ Step 1  │     │ Step 2  │     │ Step 3  │     │ Step 4  │
│ Done    │     │ Current │     │ Waiting │     │ Waiting │
└─────────┘     └─────────┘     └─────────┘     └─────────┘
   ↑              ↑                ↑                ↑
   │              │                │                └─ Wait state
   │              │                └────────────────── Wait state
   │              └───────────────────────────────── Process (current)
   └──────────────────────────────────────────────── Finish state

Vertical Layout:
┌─────────┐
│ ✓       │
│ Step 1  │
│ Done    │
└────┬────┘
     │
┌────▼────┐
│ 2       │
│ Step 2  │
│ Current │
└────┬────┘
     │
┌────▼────┐
│ 3       │
│ Step 3  │
│ Waiting │
└─────────┘
```

**Component Structure:**
- **Container**: Wrapper for all steps
- **Step Items**: Individual step indicators with icon/number
- **Connectors**: Lines between steps showing progress
- **Title**: Step name (required)
- **Description**: Additional detail (optional)
- **Status Indicator**: Visual state (number, checkmark, or error)

**Visual States:**
- **wait**: Future step, not yet started (gray)
- **process**: Current active step (highlighted)
- **finish**: Completed step (checkmark, success color)
- **error**: Failed step (error color)

## API Reference

### Props

| Prop        | Type                         | Default        | Description                                |
| ----------- | ---------------------------- | -------------- | ------------------------------------------ |
| `items`     | `StepItem[]`                 | -              | Array of step items                        |
| `current`   | `number`                     | `0`            | Current step index (0-indexed)             |
| `direction` | `'horizontal' \| 'vertical'` | `'horizontal'` | Steps direction                            |
| `size`      | `'small' \| 'default'`       | `'default'`    | Size variant                               |
| `className` | `string`                     | -              | Additional CSS class                       |
| `onChange`  | `(current: number) => void`  | -              | Step click handler (makes steps clickable) |

### StepItem

| Property      | Type                                         | Description                       |
| ------------- | -------------------------------------------- | --------------------------------- |
| `title`       | `string`                                     | Step title                        |
| `description` | `string`                                     | Step description (optional)       |
| `icon`        | `ReactNode`                                  | Custom icon (optional)            |
| `status`      | `'wait' \| 'process' \| 'finish' \| 'error'` | Manual status override (optional) |

### Direction Details

- **horizontal** (default): Steps arranged left-to-right, ideal for headers and compact spaces
- **vertical**: Steps arranged top-to-bottom, better for sidebars and detailed descriptions

### Size Details

- **small**: Compact size for dense interfaces (smaller icons and text)
- **default**: Standard size for most use cases

### Status Logic

If `status` is not manually set on a step item:
- Steps before `current` → `'finish'`
- Step at `current` → `'process'`
- Steps after `current` → `'wait'`

Manual status overrides automatic calculation.

## Best Practices

### Do ✅

- Keep titles short and meaningful (1–2 words) and use `description` for extra context
- Use the vertical `direction` for dense sidebars or long step descriptions
- Use `onChange` only when users are allowed to navigate non-linearly
- Show 3-7 steps (more may overwhelm users)
- Use descriptive titles that indicate what happens in each step

### Don't ❌

- Don't make steps clickable if earlier steps contain required validation
- Don't overload steps with long text (prefer descriptions or move detail into the page content)
- Don't rely on color alone to communicate status (pair with text and icons when relevant)
- Don't skip step numbers unless using custom icons

## Accessibility

The Steps component follows WCAG 2.1 AA guidelines:

### Built-in Features

- **Keyboard navigation**: When `onChange` is provided, steps are keyboard accessible
- **Role="button"**: Clickable steps are properly marked
- **Tab index**: Interactive steps are in tab order
- **Visual indicators**: Multiple cues beyond color (numbers, checkmarks, text)

### Keyboard Navigation

When `onChange` is provided (clickable steps):

- **Tab**: Navigate to next clickable step
- **Shift+Tab**: Navigate to previous clickable step
- **Enter/Space**: Activate step (navigate to that step)

Non-clickable steps are not in the tab order.

### Screen Reader Support

**For Basic Steps:**
```tsx
<Steps
  current={1}
  items={[
    { title: 'Login' },
    { title: 'Verify' },
    { title: 'Complete' }
  ]}
/>
// Screen reader announces status through visual position
// Consider adding aria-label for clarity
```

**With Descriptions:**
```tsx
<Steps
  current={1}
  items={[
    { title: 'Personal Info', description: 'Name and email' },
    { title: 'Payment', description: 'Billing details' },
    { title: 'Review', description: 'Confirm order' }
  ]}
/>
// Descriptions provide additional context for all users
```

**For Interactive Steps:**
```tsx
<Steps
  current={2}
  onChange={(step) => navigateToStep(step)}
  items={[
    { title: 'Step 1' },
    { title: 'Step 2' },
    { title: 'Step 3' }
  ]}
/>
// Clickable steps announce as buttons
// Screen reader: "Step 1, button", "Step 2, button", etc.
```

### Best Practices for Accessibility

#### Do ✅

- Provide descriptive step titles that make sense independently
- Use descriptions to add context for complex steps
- Ensure sufficient color contrast for all states
- Provide visual feedback beyond color (numbers, icons, text)
- Use `aria-label` to describe the overall process if needed

#### Don't ❌

- Don't rely only on color to show progress
- Don't make all steps clickable if order matters
- Don't use steps without clear visual progression
- Don't nest interactive elements inside step titles

### Color Contrast

All step states meet WCAG 2.1 AA contrast requirements:
- Text labels: Minimum 4.5:1 contrast ratio
- Icons/indicators: Minimum 3:1 contrast ratio
- Connectors: Minimum 3:1 contrast ratio

Tested in both light and dark modes.

## Common Patterns

### Checkout Flow

```tsx title="src/components/CheckoutSteps.tsx"
import { Steps } from '@grasdouble/lufa_design-system';
import { useState } from 'react';

function CheckoutSteps() {
  const [currentStep, setCurrentStep] = useState(0);

  const steps = [
    { title: 'Cart', description: 'Review items' },
    { title: 'Shipping', description: 'Delivery address' },
    { title: 'Payment', description: 'Payment method' },
    { title: 'Confirmation', description: 'Order summary' }
  ];

  return (
    <div>
      <Steps
        current={currentStep}
        items={steps}
      />
      {/* Step content here */}
      <button onClick={() => setCurrentStep(currentStep + 1)}>
        Continue
      </button>
    </div>
  );
}
```

### Onboarding Wizard

```tsx title="src/components/OnboardingWizard.tsx"
import { Steps } from '@grasdouble/lufa_design-system';
import { useState } from 'react';

function OnboardingWizard() {
  const [current, setCurrent] = useState(0);
  const [completedSteps, setCompletedSteps] = useState<number[]>([]);

  const steps = [
    { title: 'Welcome', description: 'Get started' },
    { title: 'Profile', description: 'Your information' },
    { title: 'Preferences', description: 'Customize settings' },
    { title: 'Complete', description: 'All set!' }
  ].map((step, index) => ({
    ...step,
    status: completedSteps.includes(index) ? 'finish' : undefined
  }));

  const handleStepChange = (step: number) => {
    // Only allow navigation to completed steps or next step
    if (completedSteps.includes(step) || step <= current + 1) {
      setCurrent(step);
    }
  };

  return (
    <Steps
      current={current}
      items={steps}
      onChange={handleStepChange}
    />
  );
}
```

### Error State Handling

```tsx title="src/components/StepsWithError.tsx"
import { Steps } from '@grasdouble/lufa_design-system';
import { useState } from 'react';

function StepsWithError() {
  const [current, setCurrent] = useState(1);
  const [hasError, setHasError] = useState(true);

  const steps = [
    { title: 'Information', status: 'finish' },
    { title: 'Payment', status: hasError ? 'error' : 'process' },
    { title: 'Confirmation', status: 'wait' }
  ];

  return (
    <div>
      <Steps current={current} items={steps} />
      {hasError && (
        <div className="error-message">
          Payment failed. Please check your card details.
        </div>
      )}
    </div>
  );
}
```

```tsx title="src/components/ResponsiveSteps.tsx"
function ResponsiveSteps({ current, items }) {

      items={items}
      size={isMobile ? 'small' : 'default'}
      direction={isMobile ? 'vertical' : 'horizontal'}
    />
  );
}
```

## TypeScript Support

The Steps component is fully typed with TypeScript:

```tsx
import type { ReactNode } from 'react';

interface StepItem {
  title: string;
  description?: string;
  icon?: ReactNode;
  status?: 'wait' | 'process' | 'finish' | 'error';

interface StepsProps {
  items: StepItem[];
  current?: number;
  direction?: 'horizontal' | 'vertical';
  size?: 'small' | 'default';
  className?: string;

}
```

This ensures type safety and autocomplete in your IDE.
